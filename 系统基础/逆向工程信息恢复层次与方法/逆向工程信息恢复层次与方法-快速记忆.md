# 逆向工程信息恢复层次与方法-快速记忆

## 必背结论

逆向工程导出的信息有四个层次：

```text
实现级、结构级、功能级、领域级
```

记忆口诀：

```text
实结功领
```

## 层次顺序

从低到高：

```text
实现级 < 结构级 < 功能级 < 领域级
```

通俗记忆：

```text
代码怎么写 -> 程序怎么组织 -> 系统能做什么 -> 业务上代表什么
```

## 方法对应层次

| 方法 | 对应层次 | 口诀 |
|---|---|---|
| 用户指导下的搜索与变换 | 实现级、结构级 | 用户搜变：实结 |
| 变换式方法 | 实现级、结构级、功能级 | 变换：实结功 |
| 基于领域知识的方法 | 功能级、领域级 | 领域：功领 |
| 铅板恢复法 | 实现级、结构级 | 铅板：实结 |

## 总口诀

```text
实结功领四层次
用户搜变只实结
变换方法实结功
领域知识功和领
铅板恢复也是实结
```

再压缩：

```text
四层：实结功领
方法：搜实结，变实结功，域功领，铅实结
```

## 快速判断法

### 看到“领域知识”

直接联想：

```text
功能级、领域级
```

因为有领域知识，才能理解业务领域含义。

### 看到“变换式方法”

直接联想：

```text
实现级、结构级、功能级
```

变换式方法能从程序逐步抽象到功能，但通常不到领域级。

### 看到“铅板恢复法”

直接联想：

```text
实现级、结构级
```

铅板恢复法偏底层，只记“实结”。

### 看到“用户指导下的搜索与变换”

直接联想：

```text
实现级、结构级
```

用户指导下从程序中搜索和变换，主要恢复底层实现与结构。

## 易错点

1. 变换式方法不是领域级。
2. 铅板恢复法不是功能级。
3. 基于领域知识的方法不是实现级、结构级，而是功能级、领域级。
4. 功能级和领域级不要混淆：
   - 功能级：系统能做什么。
   - 领域级：这些功能对应什么业务概念和规则。

## 最终背诵版

```text
逆向工程导出信息分四层：实现级、结构级、功能级、领域级，口诀“实结功领”。
用户指导下的搜索与变换方法导出实现级、结构级；
变换式方法导出实现级、结构级、功能级；
基于领域知识的方法导出功能级、领域级；
铅板恢复法导出实现级、结构级。
```
