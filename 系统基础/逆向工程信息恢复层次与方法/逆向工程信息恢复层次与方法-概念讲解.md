## 什么是逆向工程

逆向工程是一种通过分析现有系统，推导出系统设计原理、结构和工作机制的过程。

它的方向与正向开发相反：

```text
正向工程：需求 -> 设计 -> 编码 -> 系统
逆向工程：系统 -> 代码分析 -> 结构恢复 -> 功能理解 -> 领域理解
```

逆向工程通常不是为了直接重新开发系统，而是为了理解、维护、迁移、重构或复用已有系统。

## 逆向工程可以恢复哪些信息

逆向工程导出的信息可以分为四个抽象层次：

```text
实现级
结构级
功能级
领域级
```

这四个层次从低到高，抽象程度逐渐提高。

## 实现级

实现级关注程序实现细节。

典型内容包括：

- 程序语句。
- 变量。
- 数据结构。
- 算法。
- 控制流程。
- 具体代码实现。

通俗理解：

```text
实现级回答：代码是怎么写的？
```

例如，分析一个游戏服务端程序时，看到某个函数中如何扣除玩家金币、如何更新背包表，这属于实现级信息。

## 结构级

结构级关注程序的组织结构。

典型内容包括：

- 模块划分。
- 类和对象之间的关系。
- 函数调用关系。
- 组件关系。
- 数据流和控制流。

通俗理解：

```text
结构级回答：程序是怎么组织的？
```

例如，从已有系统中分析出账号模块、商城模块、背包模块、排行榜模块之间的调用关系，这属于结构级信息。

## 功能级

功能级关注系统对外提供的功能。

典型内容包括：

- 系统功能。
- 输入输出。
- 业务操作。
- 用户可感知的服务能力。

通俗理解：

```text
功能级回答：系统能做什么？
```

例如，分析出系统支持登录、购买道具、发放奖励、查询排行榜等能力，这属于功能级信息。

## 领域级

领域级关注系统背后的业务语义和领域规则。

典型内容包括：

- 业务概念。
- 领域对象。
- 业务规则。
- 业务流程。
- 领域模型。

通俗理解：

```text
领域级回答：这些功能在业务上代表什么？
```

例如，在游戏业务中，理解“道具”“背包”“订单”“活动奖励”“玩家资产”等领域概念，以及它们之间的业务规则，这属于领域级信息。

## 恢复信息的方法

### 用户指导下的搜索与变换方法

用户指导下的搜索与变换方法主要用于导出：

```text
实现级
结构级
```

这种方法依赖用户指导，在现有程序中进行搜索、匹配和变换，从而抽象出较低层次的信息。

### 变换式方法

变换式方法用于导出：

```text
实现级
结构级
功能级
```

它通过程序分析、表示转换、抽象和重构，逐步从代码中恢复出更高层的信息。

但它主要依赖程序本身，不一定具备领域业务知识，因此一般不认为它能直接恢复领域级信息。

### 基于领域知识的方法

基于领域知识的方法用于导出：

```text
功能级
领域级
```

这种方法利用已有的业务知识、领域规则和专家经验来理解系统。

因为它引入了领域知识，所以能够恢复领域级信息。

### 铅板恢复法

铅板恢复法用于导出：

```text
实现级
结构级
```

它主要恢复较底层的信息，关注程序实现和程序结构。

## 考试答题模板

如果考试问“逆向工程导出信息的抽象层次及恢复方法”，可以这样答：

```text
逆向工程是通过分析现有系统来推导其设计原理和工作机制的过程。
逆向工程导出的信息可分为实现级、结构级、功能级和领域级四个抽象层次。
实现级关注代码实现细节，结构级关注模块和调用关系，功能级关注系统提供的功能，领域级关注业务概念和领域规则。

常见恢复方法中，用户指导下的搜索与变换方法主要导出实现级和结构级信息；
变换式方法可导出实现级、结构级和功能级信息；
基于领域知识的方法可导出功能级和领域级信息；
铅板恢复法可导出实现级和结构级信息。
```
