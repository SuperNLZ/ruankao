## 1. 常考基础知识

### 1.1 关系模式是什么

关系模式是关系的结构描述。

形式化表示：

```text
R(U, D, dom, F)
```

速记：

```text
R：关系名
U：属性集合
D：属性域
dom：属性到域的映像
F：属性间数据依赖
```

考试中常简写为：

```text
R(A, B, C, D)
```

### 1.2 关系模型基本术语

| 概念 | 快速记忆 |
|---|---|
| 关系 | 二维表 |
| 元组 | 表中的一行 |
| 属性 | 表中的一列 |
| 域 | 属性取值范围 |
| 目/度 | 属性个数 |
| 候选码 | 唯一标识元组的最小属性集 |
| 主码 | 从候选码中选一个 |
| 主属性 | 出现在任一候选码中的属性 |
| 非主属性 | 不出现在任何候选码中的属性 |
| 外码 | 引用其他关系候选码/主码的属性 |
| 全码 | 全部属性共同构成候选码 |

### 1.3 完整性约束

```text
实体完整性：主码唯一且非空
参照完整性：外码要么为空，要么引用已存在的主码
用户定义完整性：业务规则约束
```

常见题眼：

```text
主键不能为空 -> 实体完整性
外键必须引用已有记录 -> 参照完整性
年龄、成绩、金额范围限制 -> 用户定义完整性
```

### 1.4 关系运算

传统集合运算：

```text
并、差、交、笛卡尔积
```

专门关系运算：

```text
选择、投影、连接、除
```

速记：

```text
选择 σ：选行
投影 π：选列
连接 ⋈：按条件合并两个关系
自然连接：等值连接 + 去掉重复属性列
除法 ÷：常用于“查询选修了全部课程的学生”这类题
```

### 1.5 函数依赖

函数依赖：

```text
X -> Y
```

含义：

```text
如果 X 的值相同，则 Y 的值也必然相同。
```

注意：

```text
函数依赖由语义决定，不能只看某一时刻表中的数据。
```

常见依赖：

```text
完全函数依赖：Y 依赖 X，且 X 的任何真子集都不能推出 Y
部分函数依赖：Y 依赖 X，但 X 的某个真子集也能推出 Y
传递函数依赖：X -> Y，Y -> Z，则 Z 传递依赖于 X
```

### 1.6 候选码求法

核心方法：

```text
求属性闭包 X+
```

判断：

```text
如果 X+ 包含 R 的全部属性，则 X 是超码
如果 X 不能再删减，则 X 是候选码
```

快速技巧：

```text
没有出现在任何函数依赖右部的属性，必须包含在候选码中。
```

常见闭包步骤：

```text
1. 从 X 自身开始
2. 反复使用函数依赖推出新属性
3. 直到不能推出新属性
4. 看是否覆盖全部属性
```

### 1.7 范式

范式层次：

```text
1NF -> 2NF -> 3NF -> BCNF -> 4NF -> 5NF
```

常考重点：

```text
1NF、2NF、3NF、BCNF
```

速记：

```text
1NF：属性不可再分
2NF：消除非主属性对码的部分依赖
3NF：消除非主属性对码的传递依赖
BCNF：每个决定因素都必须包含候选码
```

易错：

```text
部分依赖通常只出现在组合候选码中。
单属性候选码不会有部分依赖。
```

### 1.8 规范化的目的

规范化用于减少：

```text
数据冗余
修改异常
插入异常
删除异常
```

常见处理：

```text
通过模式分解，把低级范式转换为更高级范式。
```

### 1.9 模式分解

模式分解常考两个性质：

```text
无损连接
保持函数依赖
```

速记：

```text
无损连接：分解后再连接，不能多出或丢失原关系信息
保持函数依赖：原来的函数依赖在分解后的关系中仍能检查
```

常见判断：

```text
分解为 R1 和 R2，若 R1∩R2 能推出 R1-R2 或 R2-R1，通常可判断为无损连接。
```

### 1.10 3NF 与 BCNF 区别

3NF 更宽松，BCNF 更严格。

```text
3NF：允许某些决定因素不是候选码，只要被决定属性是主属性
BCNF：任何非平凡函数依赖 X->Y 中，X 都必须是超码
```

常见结论：

```text
BCNF 一定是 3NF
3NF 不一定是 BCNF
```

## 2. 常见题目

### 题型 1：求属性闭包

题目：

```text
R(A,B,C,D), F={A->B, B->C, C->D}，求 A+。
```

解：

```text
A+ = {A}
A->B，得到 {A,B}
B->C，得到 {A,B,C}
C->D，得到 {A,B,C,D}
```

答案：

```text
A+ = {A,B,C,D}
```

### 题型 2：求候选码

题目：

```text
R(A,B,C,D), F={A->BC, C->D}，求候选码。
```

解：

```text
A+ = {A,B,C,D}
```

答案：

```text
A
```

### 题型 3：多个属性必须进码

题目：

```text
R(A,B,C,D), F={B->C, C->D}，求候选码。
```

解：

```text
A 和 B 没有出现在右部，必须进候选码。
AB+ = {A,B,C,D}
```

答案：

```text
AB
```

### 题型 4：判断 2NF

题目：

```text
R(A,B,C), 候选码为 AB，存在 A->C。
```

解：

```text
C 是非主属性
C 依赖于组合码 AB 的一部分 A
```

答案：

```text
不满足 2NF
```

### 题型 5：判断 3NF

题目：

```text
R(A,B,C), A->B, B->C，A 是候选码。
```

解：

```text
A -> B -> C
C 传递依赖于候选码 A
```

答案：

```text
不满足 3NF
```

### 题型 6：3NF 但不是 BCNF

题目：

```text
R(A,B,C), F={AB->C, C->B}
```

分析：

```text
候选码：AB、AC
B 和 C 都是主属性
C->B 中 C 不是超码
```

答案：

```text
满足 3NF，但不满足 BCNF
```

### 题型 7：无损连接判断

题目：

```text
R(A,B,C), F={A->B}，分解为 R1(A,B), R2(A,C)，是否无损？
```

解：

```text
R1∩R2 = {A}
A -> B，即 A 能推出 R1-R2
```

答案：

```text
无损连接
```

### 题型 8：关系运算判断

题目：

```text
从关系 R 中选出年龄大于 20 的元组，属于什么运算？
```

答案：

```text
选择运算
```

### 题型 9：投影运算

题目：

```text
只取学生关系中的学号、姓名两列，属于什么运算？
```

答案：

```text
投影运算
```

### 题型 10：自然连接

题目：

```text
自然连接与等值连接的区别是什么？
```

答案：

```text
自然连接要求同名属性相等，并去掉重复属性列。
```

## 3. 预测题目

### 预测 1：候选码基础

```text
R(A,B,C,D,E), F={A->BC, C->D, D->E}，候选码是？
```

答案：

```text
A
```

理由：

```text
A+ = {A,B,C,D,E}
```

### 预测 2：复合候选码

```text
R(A,B,C,D), F={A->C, B->D}，候选码是？
```

答案：

```text
AB
```

理由：

```text
A 和 B 都未出现在右部，必须进码；AB+ = {A,B,C,D}
```

### 预测 3：部分依赖

```text
R(A,B,C,D), 候选码为 AB，F={A->C, AB->D}，最高满足哪级范式？
```

答案：

```text
1NF，不满足 2NF
```

理由：

```text
C 是非主属性，且 C 部分依赖于组合码 AB 的一部分 A。
```

### 预测 4：传递依赖

```text
R(A,B,C,D), A->B, B->C, A->D，A 为候选码，是否满足 3NF？
```

答案：

```text
不满足 3NF
```

理由：

```text
C 传递依赖于 A。
```

### 预测 5：BCNF 判断

```text
R(A,B,C), F={A->B, B->A, A->C}，是否满足 BCNF？
```

答案：

```text
满足 BCNF
```

理由：

```text
A 和 B 都是候选码，每个决定因素都是超码。
```

### 预测 6：3NF 非 BCNF

```text
R(A,B,C), F={AB->C, C->B}，最高满足哪级范式？
```

答案：

```text
3NF，但不满足 BCNF
```

理由：

```text
C->B 中 C 不是超码，但 B 是主属性。
```

### 预测 7：无损连接

```text
R(A,B,C), F={B->C}，分解为 R1(A,B), R2(B,C)，是否无损？
```

答案：

```text
无损
```

理由：

```text
R1∩R2={B}，B->C，可推出 R2-R1。
```

### 预测 8：有损分解

```text
R(A,B,C), F={A->B}，分解为 R1(A,C), R2(B,C)，是否一定无损？
```

答案：

```text
不一定无损
```

理由：

```text
交集是 C，C 不能推出 A 或 B。
```

### 预测 9：关系运算

```text
查询同时选修了全部必修课的学生，关系代数中常用什么运算？
```

答案：

```text
除法运算
```

### 预测 10：完整性约束

```text
员工表中的部门号必须来自部门表中已存在的部门号，属于什么完整性？
```

答案：

```text
参照完整性
```

## 4. 最终背诵版

```text
关系模式是关系结构的描述，常记为 R(A,B,C...)。
关系模型中，关系是表，元组是行，属性是列，候选码是唯一标识元组的最小属性集。
函数依赖 X->Y 表示 X 能决定 Y，候选码通过属性闭包求解。
没出现在依赖右部的属性必须包含在候选码中。
1NF 要求属性原子；2NF 消除非主属性对码的部分依赖；3NF 消除非主属性对码的传递依赖；BCNF 要求每个决定因素都是超码。
模式分解重点看无损连接和保持函数依赖。
选择是选行，投影是选列，自然连接会去掉重复属性列，除法常用于“全部”类查询。
```

## 5. 易错点

1. 函数依赖由语义决定，不能只根据某一时刻的数据判断。
2. 候选码必须是最小超码，不能包含多余属性。
3. 没出现在右部的属性通常必须进候选码。
4. 2NF 主要检查部分依赖，通常和组合码有关。
5. 3NF 检查传递依赖。
6. BCNF 比 3NF 更严格。
7. 3NF 不一定是 BCNF，但 BCNF 一定是 3NF。
8. 无损连接不等于保持函数依赖，两者是不同性质。
