## 1. 常考基础知识

### 1.1 数据模型三要素

```text
数据结构 + 数据操作 + 完整性约束
```

常见数据模型：

```text
层次模型、网状模型、关系模型、面向对象模型、半结构化模型
```

考试重点通常在关系模型。

### 1.2 数据库三级模式

```text
外模式 -> 概念模式 -> 内模式
```

速记：

```text
外模式：用户视图，多个
概念模式：全局逻辑结构，一个
内模式：物理存储结构，一个
```

两级映像：

```text
外模式/概念模式映像：保证逻辑数据独立性
概念模式/内模式映像：保证物理数据独立性
```

### 1.3 DBMS 主要功能

```text
数据定义
数据操纵
数据库运行管理
数据组织、存储和管理
数据库建立和维护
```

常考关键词：

```text
并发控制、安全性控制、完整性检查、事务管理、故障恢复、日志管理
```

### 1.4 关系模型基本概念

| 概念 | 速记 |
|---|---|
| 关系 | 二维表 |
| 元组 | 表中的一行 |
| 属性 | 表中的一列 |
| 域 | 属性取值范围 |
| 候选码 | 能唯一标识元组的最小属性集 |
| 主码 | 从候选码中选出的一个 |
| 外码 | 引用其他关系主码的属性 |
| 主属性 | 包含在任一候选码中的属性 |
| 非主属性 | 不包含在任何候选码中的属性 |

### 1.5 完整性约束

```text
实体完整性：主码不能为空
参照完整性：外码要么为空，要么引用已存在的主码
用户定义完整性：业务规则约束
```

例子：

```text
成绩必须在 0~100 之间，就是用户定义完整性。
```

### 1.6 函数依赖与候选码

函数依赖：

```text
X -> Y
表示 X 的值确定后，Y 的值也能唯一确定。
```

候选码求法：

```text
求属性闭包 X+
如果 X+ 包含全部属性，则 X 是超码
如果 X 不能再删减，则 X 是候选码
```

快速技巧：

```text
没有出现在任何函数依赖右部的属性，通常必须包含在候选码中。
```

### 1.7 范式

范式层次：

```text
1NF -> 2NF -> 3NF -> BCNF -> 4NF -> 5NF
```

常考到 3NF 和 BCNF。

速记：

```text
1NF：属性不可再分，字段原子性
2NF：消除非主属性对候选码的部分依赖
3NF：消除非主属性对候选码的传递依赖
BCNF：每个决定因素都必须包含候选码
```

易错：

```text
只有组合码才容易出现部分依赖。
单属性码不会有部分依赖。
```

### 1.8 数据库设计阶段

常见阶段：

```text
需求分析
概念结构设计
逻辑结构设计
物理结构设计
应用程序设计
运行维护
```

对应产物：

```text
需求分析：需求说明
概念结构设计：E-R 图
逻辑结构设计：关系模式
物理结构设计：索引、存储结构、存取路径
```

### 1.9 E-R 图

```text
矩形：实体
椭圆：属性
菱形：联系
下画线属性：主码
```

联系类型：

```text
1:1
1:n
m:n
```

E-R 转关系：

```text
实体 -> 一个关系模式
1:1 联系 -> 可并入任一端
1:n 联系 -> 通常把 1 端主码放入 n 端作为外码
m:n 联系 -> 单独转换为一个关系模式，包含两端主码和联系属性
```

### 1.10 事务 ACID

```text
A Atomicity 原子性：事务要么全做，要么全不做
C Consistency 一致性：事务前后数据库保持一致状态
I Isolation 隔离性：并发事务之间互不干扰
D Durability 持久性：提交后的结果永久保存
```

常见并发问题：

```text
丢失更新
脏读
不可重复读
幻读
```

### 1.11 数据仓库与 OLAP

数据仓库特点：

```text
面向主题
集成
相对稳定
反映历史变化
```

OLTP 与 OLAP：

```text
OLTP：面向事务处理，增删改查频繁
OLAP：面向分析决策，查询和统计为主
```

### 1.12 NoSQL

NoSQL 常见类型：

```text
键值数据库
文档数据库
列族数据库
图数据库
```

常考判断：

```text
高并发、海量数据、分布式、弱结构化数据 -> 可能选 NoSQL
强事务、复杂关联、严格一致性 -> 优先关系数据库
```

## 2. 常考 SQL 语句

### 2.1 DDL：定义数据结构

建表：

```sql
CREATE TABLE Student (
  Sno CHAR(10) PRIMARY KEY,
  Sname VARCHAR(50) NOT NULL,
  Age INT,
  Dept VARCHAR(50)
);
```

外键：

```sql
CREATE TABLE Score (
  Sno CHAR(10),
  Cno CHAR(10),
  Grade INT,
  PRIMARY KEY (Sno, Cno),
  FOREIGN KEY (Sno) REFERENCES Student(Sno)
);
```

修改表：

```sql
ALTER TABLE Student ADD Email VARCHAR(100);
```

删除表：

```sql
DROP TABLE Student;
```

### 2.2 DML：增删改查

插入：

```sql
INSERT INTO Student(Sno, Sname, Age, Dept)
VALUES ('S01', '张三', 20, 'CS');
```

更新：

```sql
UPDATE Student
SET Dept = 'SE'
WHERE Sno = 'S01';
```

删除：

```sql
DELETE FROM Student
WHERE Sno = 'S01';
```

查询：

```sql
SELECT Sno, Sname
FROM Student
WHERE Dept = 'CS';
```

### 2.3 条件查询

```sql
SELECT *
FROM Student
WHERE Age BETWEEN 18 AND 22;
```

```sql
SELECT *
FROM Student
WHERE Sname LIKE '张%';
```

```sql
SELECT *
FROM Student
WHERE Dept IN ('CS', 'SE');
```

### 2.4 排序

```sql
SELECT *
FROM Student
ORDER BY Age DESC;
```

### 2.5 聚合与分组

常见聚合函数：

```text
COUNT、SUM、AVG、MAX、MIN
```

分组统计：

```sql
SELECT Dept, COUNT(*) AS Num
FROM Student
GROUP BY Dept;
```

分组后筛选：

```sql
SELECT Dept, COUNT(*) AS Num
FROM Student
GROUP BY Dept
HAVING COUNT(*) > 10;
```

易错：

```text
WHERE 在分组前筛选行
HAVING 在分组后筛选组
```

### 2.6 连接查询

内连接：

```sql
SELECT S.Sno, S.Sname, C.Cname, SC.Grade
FROM Student S
JOIN SC ON S.Sno = SC.Sno
JOIN Course C ON SC.Cno = C.Cno;
```

左外连接：

```sql
SELECT S.Sno, S.Sname, SC.Grade
FROM Student S
LEFT JOIN SC ON S.Sno = SC.Sno;
```

速记：

```text
INNER JOIN：只保留匹配记录
LEFT JOIN：保留左表全部记录
RIGHT JOIN：保留右表全部记录
```

### 2.7 子查询

IN：

```sql
SELECT Sname
FROM Student
WHERE Sno IN (
  SELECT Sno
  FROM SC
  WHERE Cno = 'C01'
);
```

EXISTS：

```sql
SELECT Sname
FROM Student S
WHERE EXISTS (
  SELECT 1
  FROM SC
  WHERE SC.Sno = S.Sno
);
```

比较子查询：

```sql
SELECT Sname
FROM Student
WHERE Age > (
  SELECT AVG(Age)
  FROM Student
);
```

### 2.8 DCL 与事务控制

授权：

```sql
GRANT SELECT ON Student TO user1;
```

撤销权限：

```sql
REVOKE SELECT ON Student FROM user1;
```

提交：

```sql
COMMIT;
```

回滚：

```sql
ROLLBACK;
```

## 3. 常见题目

### 题型 1：求候选码

题目：

```text
R(A,B,C,D), F={A->BC, C->D}，求候选码。
```

解法：

```text
A+ = {A}
A -> BC，所以 A+ = {A,B,C}
C -> D，所以 A+ = {A,B,C,D}
```

答案：

```text
A 是候选码
```

### 题型 2：判断范式

题目：

```text
R(A,B,C), 候选码为 AB，存在 A->C。
```

判断：

```text
C 是非主属性
C 部分依赖于组合码 AB 的一部分 A
```

答案：

```text
不满足 2NF
```

### 题型 3：传递依赖

题目：

```text
R(A,B,C), A->B, B->C，A 是候选码。
```

判断：

```text
A -> B -> C
C 传递依赖于候选码 A
```

答案：

```text
不满足 3NF
```

### 题型 4：E-R 图转关系

题目：

```text
学生和课程是 m:n 联系，联系有属性“成绩”。
```

答案：

```text
学生(学号, 姓名, ...)
课程(课程号, 课程名, ...)
选修(学号, 课程号, 成绩)
```

其中：

```text
选修关系的主码通常是 (学号, 课程号)
```

### 题型 5：SQL 分组查询

题目：

```text
查询每个部门人数超过 10 的部门编号和人数。
```

答案：

```sql
SELECT Dept, COUNT(*) AS Num
FROM Student
GROUP BY Dept
HAVING COUNT(*) > 10;
```

易错：

```text
不能把 COUNT(*) > 10 写在 WHERE 中。
```

### 题型 6：SQL 连接查询

题目：

```text
查询学生姓名、课程名和成绩。
```

答案：

```sql
SELECT S.Sname, C.Cname, SC.Grade
FROM Student S
JOIN SC ON S.Sno = SC.Sno
JOIN Course C ON SC.Cno = C.Cno;
```

### 题型 7：三级模式与数据独立性

题目：

```text
改变索引结构但不影响表的逻辑结构，体现了什么独立性？
```

答案：

```text
物理数据独立性
```

### 题型 8：OLTP 与 OLAP

题目：

```text
银行转账、订单支付属于 OLTP 还是 OLAP？
```

答案：

```text
OLTP
```

## 4. 预测题目

### 预测 1：候选码闭包

```text
R(A,B,C,D,E), F={A->BC, C->D, D->E}，R 的候选码是？
```

参考答案：

```text
A
```

理由：

```text
A+ = {A,B,C,D,E}
```

### 预测 2：右部未出现属性

```text
R(A,B,C,D), F={B->C, C->D}，候选码必须包含哪个属性？
```

参考答案：

```text
A 和 B
```

理由：

```text
A、B 没有出现在任何依赖右部，必须包含在候选码中。
AB+ = {A,B,C,D}
```

### 预测 3：范式判断

```text
R(A,B,C,D), 候选码为 AB，存在 A->C，AB->D。该关系最高满足哪一级范式？
```

参考答案：

```text
1NF，但不满足 2NF
```

理由：

```text
C 部分依赖于组合码 AB 的一部分 A。
```

### 预测 4：3NF 判断

```text
R(A,B,C), A->B, B->C，A 为候选码。该关系是否满足 3NF？
```

参考答案：

```text
不满足 3NF
```

理由：

```text
C 通过 B 传递依赖于 A。
```

### 预测 5：BCNF 判断

```text
R(A,B,C), F={A->B, B->A, A->C}，判断是否满足 BCNF。
```

参考答案：

```text
满足 BCNF
```

理由：

```text
A 和 B 都可以推出全部属性，都是候选码；每个决定因素都包含候选码。
```

### 预测 6：E-R 转关系

```text
教师和课程是 1:n 联系，一个教师可讲多门课程，一门课程只能由一个教师讲授。
```

参考答案：

```text
教师(教师号, ...)
课程(课程号, ..., 教师号)
```

理由：

```text
1:n 联系通常把 1 端主码加入 n 端作为外码。
```

### 预测 7：SQL HAVING

```text
查询平均成绩大于 80 的课程号和平均成绩。
```

参考答案：

```sql
SELECT Cno, AVG(Grade) AS AvgGrade
FROM SC
GROUP BY Cno
HAVING AVG(Grade) > 80;
```

### 预测 8：SQL 子查询

```text
查询选修了 C01 课程的学生姓名。
```

参考答案：

```sql
SELECT Sname
FROM Student
WHERE Sno IN (
  SELECT Sno
  FROM SC
  WHERE Cno = 'C01'
);
```

### 预测 9：事务特性

```text
事务提交后，即使系统故障，修改结果也不应丢失。这体现 ACID 中的哪一项？
```

参考答案：

```text
持久性 Durability
```

### 预测 10：NoSQL 判断

```text
需要存储海量非结构化文档，并支持水平扩展，应优先考虑哪类数据库？
```

参考答案：

```text
文档型 NoSQL 数据库
```

## 5. 最终背诵版

``` text
数据库常考点集中在：三级模式、关系模型、完整性约束、候选码、函数依赖、范式、E-R 图、SQL、事务和 NoSQL。
三级模式是外模式、概念模式、内模式；两级映像保证逻辑和物理数据独立性。
关系模型中，关系是表，元组是行，属性是列，候选码是唯一标识元组的最小属性集。
范式中，1NF 要求属性原子，2NF 消除部分依赖，3NF 消除传递依赖，BCNF 要求决定因素包含候选码。
SQL 中 WHERE 先筛选行，GROUP BY 分组，HAVING 筛选组，ORDER BY 排序。
E-R 转关系时，实体转表，1:n 把 1 端主码放入 n 端，m:n 单独成表。
```
