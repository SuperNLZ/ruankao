# SOA 架构-快速记忆

## 一句话

```text
SOA = 把业务能力封装成服务，供多个系统复用。
```

## 核心关键词

```text
服务
松耦合
标准接口
服务复用
服务注册/发现
服务组合
ESB
```

## 三个角色

```text
服务提供者：提供服务
服务消费者：调用服务
服务注册中心：发布、查找、发现服务
```

## 最短口诀

```text
服务封装，松耦复用；
注册发现，组合治理。
```

## 看到这些词就选 SOA

| 关键词 | 判断 |
|---|---|
| 面向服务 | SOA |
| 服务复用 | SOA |
| 服务注册/发现 | SOA |
| 服务提供者/消费者 | SOA |
| ESB 企业服务总线 | SOA |
| 异构系统集成 | SOA |
| 服务组合/服务编排 | SOA |

## SOA vs 微服务

```text
SOA：偏企业系统集成，服务粒度较粗，常见 ESB。
微服务：偏系统内部拆分，服务粒度较细，强调独立部署。
```

## 易错点

```text
SOA 不是一种具体技术，而是一种架构思想。
SOA 不等于 Web Service，Web Service 只是实现 SOA 的一种方式。
SOA 强调服务复用和系统集成。
微服务强调小粒度拆分和独立部署。
```

## 最终背诵版

```text
SOA 是面向服务架构，将系统功能封装为可复用、松耦合、标准化的服务。
SOA 的核心是服务提供者、服务消费者和服务注册中心。
它强调服务封装、松耦合、标准接口、服务复用、服务发现和服务组合，
常用于企业异构系统集成。
```
