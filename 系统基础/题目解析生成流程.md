# 题目解析生成流程

后续根据截图生成题目解析时，统一按下面流程执行，避免只写了 Markdown 或脚本但没有真正生成 PDF 和图片。

## 标准流程

```text
创建目录
写 Markdown
写/复用生成脚本
执行生成脚本
find 检查文件
检查 PDF 大小和页数
最后再回复
```

## 目录规则

每道题按题目名称单独建立目录：

```text
系统基础/
  题目名称/
    题目名称解题过程.md
    题目名称解题过程.pdf
    images/
      题目名称-题目截图.png
      题目名称-问题1.png
      题目名称-问题2.png
      ...
    generate_xxx_pdf.py
```

如果题目不属于“系统基础”，则按对应知识模块新建一级目录。

## 完成标准

在最终回复用户前，必须确认以下内容：

1. 题目目录已经创建。
2. Markdown 解题过程已经生成。
3. 图片目录 `images/` 已经生成。
4. 题目图、问题图或参考答案图已经生成到 `images/`。
5. PDF 已经生成到题目目录。
6. 已用 `find` 检查目录下文件是否齐全。
7. 已检查 PDF 文件大小。
8. 已检查 PDF 页数。

## 回复前检查命令

```bash
find 系统基础/题目名称 -maxdepth 2 -type f
ls -lh 系统基础/题目名称/题目名称解题过程.pdf
python3 -c "from pypdf import PdfReader; print(len(PdfReader('系统基础/题目名称/题目名称解题过程.pdf').pages))"
```

## 注意事项

- 不能在生成脚本执行前说“已生成 PDF”。
- 不能只创建 Markdown 和脚本就结束。
- 如果截图无法直接落盘，使用截图内容重绘复习用图片，并嵌入 PDF。
- 最终回复中要给出 Markdown 和 PDF 的文件链接，并说明 PDF 页数和大小。
