# 题目解析生成流程

后续根据截图生成题目解析时，统一按下面流程执行，避免只写了 Markdown 或脚本但没有真正生成 PDF 和图片。

## 标准流程

```text
创建目录
写 Markdown
写/复用生成脚本
执行生成脚本
find 检查文件
检查 PDF 大小和页数
最后再回复
```

## 目录规则

每道题按题目名称单独建立目录：

```text
系统基础/
  题目名称/
    题目名称-快速记忆.md
    题目名称-快速记忆.pdf
    images/
      题目名称-题目截图.png
      题目名称-问题1.png
      题目名称-问题2.png
      ...
    generate_xxx_pdf.py
```

如果题目不属于“系统基础”，则按对应知识模块新建一级目录。

## 版本生成规则

默认只生成“快速记忆”版本：

```text
题目或知识点名称-快速记忆.md
题目或知识点名称-快速记忆.pdf
```

只有当用户明确要求“两个版本”“概念讲解 + 快速记忆”“都生成”时，才同时生成：

```text
题目或知识点名称-概念讲解.md
题目或知识点名称-快速记忆.md
题目或知识点名称-概念讲解.pdf
题目或知识点名称-快速记忆.pdf
```

如果用户明确要求“详细解题过程”“完整推导过程”，可以生成：

```text
题目名称解题过程.md
题目名称解题过程.pdf
```

但用户没有特别说明时，默认按“快速记忆”生成。

## 完成标准

在最终回复用户前，必须确认以下内容：

1. 题目目录已经创建。
2. Markdown 文档已经生成。
3. 图片目录 `images/` 已经生成。
4. 题目图、问题图或参考答案图已经生成到 `images/`。
5. PDF 已经生成到题目目录。
6. 已用 `find` 检查目录下文件是否齐全。
7. 已检查 PDF 文件大小。
8. 已检查 PDF 页数。

## 回复前检查命令

```bash
find 系统基础/题目名称 -maxdepth 2 -type f
ls -lh 系统基础/题目名称/题目名称-快速记忆.pdf
python3 -c "from pypdf import PdfReader; print(len(PdfReader('系统基础/题目名称/题目名称-快速记忆.pdf').pages))"
```

## 注意事项

- 不能在生成脚本执行前说“已生成 PDF”。
- 不能只创建 Markdown 和脚本就结束。
- 如果截图无法直接落盘，使用截图内容重绘复习用图片，并嵌入 PDF。
- 最终回复中要给出 Markdown 和 PDF 的文件链接，并说明 PDF 页数和大小。

## 案例分析题目规则

如果用户提供的是案例分析题，生成文档时必须把题目的原文描述加入 Markdown 和 PDF，至少包括：

```text
试题编号和分值
说明原文
问题原文
题图说明
```

案例题文档建议采用下面结构：

```text
题目原文
题图
问题结论
解题思路
标准答题模板
关键词速记
易错点
```

如果用户分多次补充问题，后续更新同一个题目目录下的文档和 PDF，不要另起重复目录。

## 概念类文档规则

如果用户要求整理某个“概念、知识点、架构、方法、区别、优缺点”等非题目解析内容，也遵守上面的版本生成规则。

默认只生成：

```text
题目或知识点名称-快速记忆.md
题目或知识点名称-快速记忆.pdf
```

只有用户明确要求两个版本时，才拆分为两个 Markdown 文档：

```text
题目或知识点名称-概念讲解.md
题目或知识点名称-快速记忆.md
```

其中：

1. 概念讲解文档：负责讲清楚定义、背景、组成、原理、应用场景、例子、优缺点、考试答题模板。
2. 快速记忆文档：负责口诀、关键词表、易错点、选择题快速判断法、最终背诵版。内容必须压缩、极简，优先控制在 1-2 页 PDF，不展开长解释。

如果生成两个版本，PDF 也分别生成：

```text
题目或知识点名称-概念讲解.pdf
题目或知识点名称-快速记忆.pdf
```

## 快速记忆版写法

快速记忆版必须尽量简化，优先使用：

```text
一句话结论
口诀
关键词 -> 答案
易混点
最终背诵版
```

避免大段背景、长篇定义、完整推导和过多例子。
