# Kruchten 4+1 视图模型-快速记忆

## 1. 一句话

Kruchten 的“4+1”视图模型用 5 个视角描述软件架构：

```text
逻辑视图、开发视图、进程视图、物理视图 + 场景视图
```

其中，场景视图用于串联和验证前四个视图。

## 2. 口诀

```text
逻辑看功能，
开发看代码，
进程看运行，
物理看部署，
场景看用例验证。
```

## 3. 五个视图速记表

| 视图 | 关注点 | 常见关键词 |
|---|---|---|
| 逻辑视图 | 系统功能和对象模型 | 功能、类、对象、业务逻辑 |
| 开发视图 | 软件模块和代码组织 | 模块、包、组件、分层、依赖 |
| 进程视图 | 运行时行为 | 进程、线程、并发、通信、性能 |
| 物理视图 | 部署结构 | 服务器、节点、网络、部署、拓扑 |
| 场景视图 | 用例验证 | 用例、场景、关键流程、架构验证 |

## 4. 常见别名

```text
开发视图 = 实现视图
物理视图 = 部署视图
场景视图 = 用例视图
```

## 5. 选择题快速判断

看到“功能、类、对象、业务逻辑”：

```text
选逻辑视图
```

看到“模块、包、组件、代码组织、分层”：

```text
选开发视图
```

看到“进程、线程、并发、同步、通信、性能”：

```text
选进程视图
```

看到“服务器、节点、网络、部署、拓扑”：

```text
选物理视图
```

看到“用例、场景、关键流程、验证架构”：

```text
选场景视图
```

## 6. 最终背诵版

```text
Kruchten 的 4+1 视图模型包括逻辑视图、开发视图、进程视图、物理视图和场景视图。
逻辑视图关注系统功能和对象模型；
开发视图关注模块、组件和代码组织；
进程视图关注运行时进程、线程、通信和性能；
物理视图关注软件到硬件节点的部署；
场景视图通过用例或典型场景串联并验证前四个视图。
```

## 7. 易错点

1. “4+1”不是 4 个视图，而是 5 个视角。
2. “+1”是场景视图，也常称用例视图。
3. 开发视图不是讲开发流程，而是讲代码、模块、组件组织。
4. 进程视图不是操作系统进程知识点，而是架构运行时并发和通信视角。
5. 物理视图重点是部署节点和网络拓扑，不是物理存储结构。
