# 数据库范式-快速记忆

## 一句话

```text
数据库范式 = 用拆表消除数据冗余、插入异常、删除异常、修改异常。
```

## 先背两个词

```text
候选码：能唯一标识一行，并且最小。
非主属性：不属于任何候选码的属性。
```

考试判断范式，核心看：

```text
部分依赖、传递依赖、决定因素、多值依赖
```

## 范式口诀

```text
一原子，二完全，三传递，BCNF看决定，四多值
```

对应关系：

```text
1NF    -> 属性不可再分
2NF    -> 非主属性完全依赖候选码
3NF    -> 非主属性不能传递依赖候选码
BCNF   -> 每个决定因素都必须是候选码/超码
4NF    -> 消除非平凡多值依赖
```

## 1NF：属性原子

关键词：

```text
字段不可再分
不能一个单元格存多个值
```

例子：

```text
电话 = "138...,139..." 违反 1NF
拆成多行或单独电话表
```

## 2NF：消除部分依赖

前提：

```text
已经满足 1NF
通常出现在“组合候选码”中
```

判断：

```text
非主属性只依赖组合码的一部分 -> 违反 2NF
```

例子：

```text
选课(学号, 课程号, 学生名, 课程名, 成绩)
候选码 = (学号, 课程号)

学生名 只依赖 学号
课程名 只依赖 课程号
成绩 依赖 (学号, 课程号)

所以违反 2NF
```

拆表：

```text
学生(学号, 学生名)
课程(课程号, 课程名)
选课(学号, 课程号, 成绩)
```

## 3NF：消除传递依赖

前提：

```text
已经满足 2NF
```

判断：

```text
候选码 -> 非主属性A -> 非主属性B
则 B 传递依赖候选码，违反 3NF
```

例子：

```text
学生(学号, 姓名, 系号, 系名)

学号 -> 系号
系号 -> 系名
所以 学号 -> 系名 是传递依赖
违反 3NF
```

拆表：

```text
学生(学号, 姓名, 系号)
院系(系号, 系名)
```

## BCNF：决定因素必须是码

前提：

```text
比 3NF 更严格
```

判断：

```text
只要存在 X -> Y
X 不是候选码/超码
就违反 BCNF
```

记忆：

```text
3NF 看非主属性
BCNF 看所有决定因素
```

## 4NF：消除多值依赖

关键词：

```text
一个实体有两组互相独立的多值属性
多值依赖 X ->> Y
```

判断：

```text
存在非平凡多值依赖 X ->> Y
且 X 不是候选码/超码
就违反 4NF
```

例子：

```text
学生爱好电话(学号, 爱好, 电话)

一个学生有多个爱好
一个学生有多个电话
爱好和电话互相独立
```

会被迫组合成：

```text
张三, 篮球, 138
张三, 篮球, 139
张三, 游泳, 138
张三, 游泳, 139
```

拆表：

```text
学生爱好(学号, 爱好)
学生电话(学号, 电话)
```

记忆：

```text
1NF：一格不能多个值
4NF：一表不能混多个独立多值事实
```

## 高频选择题判断

```text
属性不可分                 -> 1NF
部分函数依赖               -> 违反 2NF
传递函数依赖               -> 违反 3NF
决定因素不是候选码/超码     -> 违反 BCNF
非平凡多值依赖             -> 违反 4NF
消除冗余和更新异常         -> 范式化目的
```

## 最终背诵版

```text
一原子：1NF 要求属性不可再分。
二完全：2NF 消除非主属性对组合码的部分依赖。
三传递：3NF 消除非主属性对候选码的传递依赖。
BCNF看决定：每个函数依赖的决定因素都必须是候选码或超码。
四多值：4NF 消除非平凡多值依赖。
```
