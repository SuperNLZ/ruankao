# ISO/IEC 25010 软件质量模型-快速记忆

## 一句话

```text
ISO/IEC 25010 = 软件质量评价模型。
```

软考常考：

```text
2011 版产品质量模型 8 大特性。
```

## 八大特性口诀

```text
功性兼用，可安维移
```

展开：

```text
功能适合性
性能效率
兼容性
易用性
可靠性
安全性
可维护性
可移植性
```

## 关键词速记

| 特性 | 看到这些词就选 |
|---|---|
| 功能适合性 | 功能完整、功能正确、满足需求 |
| 性能效率 | 响应时间、资源利用、容量、并发 |
| 兼容性 | 共存、互操作、和其他系统协同 |
| 易用性 | 易学、易操作、界面、误操作防护 |
| 可靠性 | 可用、容错、恢复、稳定运行 |
| 安全性 | 保密、完整、防抵赖、认证、审计 |
| 可维护性 | 模块化、易修改、易测试、易分析 |
| 可移植性 | 适配、安装、迁移、替换 |

## 最容易混的 4 组

### 可靠性 vs 安全性

```text
可靠性：系统别挂，挂了能恢复。
安全性：数据别泄露，操作别被伪造。
```

### 兼容性 vs 可移植性

```text
兼容性：和别人一起工作。
可移植性：自己换环境运行。
```

### 性能效率 vs 可靠性

```text
性能效率：快不快、撑不撑得住。
可靠性：稳不稳、坏了能不能恢复。
```

### 可维护性 vs 可移植性

```text
可维护性：改代码方便。
可移植性：换环境方便。
```

## 2023 版提醒

新版 ISO/IEC 25010:2023 产品质量模型是 9 个特性。

考试如果出现：

```text
交互能力
灵活性
Safety
```

要注意可能是在考新版。

但软考常规题优先记：

```text
功性兼用，可安维移
```
