# ISO/IEC 25010 软件质量模型-概念讲解

## 版本说明

ISO/IEC 25010 属于 SQuaRE 系列标准，用于描述系统和软件产品质量模型。

软考题库和教材中常见的是 ISO/IEC 25010:2011 的表达，尤其常考：

```text
8 个产品质量特性
```

需要注意的是，ISO/IEC 25010 已有 2023 版。新版产品质量模型调整为 9 个特性。

备考时建议：

```text
软考选择题优先按常见 2011 版八大特性记忆；
如果题干或选项出现“交互能力、灵活性、安全保障 Safety”等新版词汇，再按 2023 版判断。
```

## ISO/IEC 25010 是什么

ISO/IEC 25010 是软件质量模型标准，用于描述和评价软件产品质量。

它回答的问题是：

```text
一个软件系统到底好不好，可以从哪些质量维度评价？
```

例如，一个手机游戏服务端不仅要功能正确，还要性能好、稳定、安全、容易维护、容易部署迁移。

这些都属于软件质量评价范围。

## 2011 版常考：产品质量模型 8 大特性

ISO/IEC 25010:2011 中的软件产品质量模型常见 8 个质量特性：

```text
功能适合性
性能效率
兼容性
易用性
可靠性
安全性
可维护性
可移植性
```

### 1. 功能适合性

功能适合性关注软件提供的功能是否满足明确或隐含的需求。

常见子特性：

- 功能完整性。
- 功能正确性。
- 功能适当性。

游戏样例：

```text
商城系统是否完整支持商品展示、购买、扣款、发放道具、订单记录。
```

如果功能缺失或计算错误，就是功能适合性问题。

### 2. 性能效率

性能效率关注系统在指定条件下，使用资源完成任务的效率。

常见子特性：

- 时间特性。
- 资源利用率。
- 容量。

游戏样例：

```text
活动开启时，大量玩家同时请求排行榜，系统是否能在可接受时间内返回结果。
```

响应慢、CPU/内存消耗过高、并发容量不足，都属于性能效率问题。

### 3. 兼容性

兼容性关注软件能否与其他系统、产品或组件共存、协同工作。

常见子特性：

- 共存性。
- 互操作性。

游戏样例：

```text
游戏客户端能否兼容不同渠道 SDK；
服务端能否与支付平台、登录平台、数据统计平台正确交互。
```

### 4. 易用性

易用性关注用户在指定场景下使用软件是否容易理解、学习和操作。

常见子特性：

- 可识别性。
- 易学性。
- 易操作性。
- 用户差错防护。
- 用户界面美观性。
- 可访问性。

游戏样例：

```text
玩家是否容易找到购买入口；
错误提示是否清晰；
误操作是否有二次确认。
```

### 5. 可靠性

可靠性关注系统在指定条件和时间内持续正确运行的能力。

常见子特性：

- 成熟性。
- 可用性。
- 容错性。
- 可恢复性。

游戏样例：

```text
服务器节点故障后，玩家是否还能登录；
服务异常后是否能恢复玩家订单和背包数据。
```

宕机、数据丢失、故障无法恢复，都属于可靠性问题。

### 6. 安全性

安全性关注系统保护信息和数据的能力，防止未授权访问、篡改和抵赖。

常见子特性：

- 保密性。
- 完整性。
- 不可抵赖性。
- 可追责性。
- 真实性。

游戏样例：

```text
接口签名防篡改；
支付回调校验；
账号登录鉴权；
关键操作记录审计日志。
```

### 7. 可维护性

可维护性关注软件被分析、修改、测试和复用的难易程度。

常见子特性：

- 模块化。
- 可重用性。
- 可分析性。
- 可修改性。
- 可测试性。

游戏样例：

```text
活动系统是否模块化；
新增节日活动时是否只需增加配置和少量代码；
问题出现后是否容易定位。
```

### 8. 可移植性

可移植性关注软件从一个环境迁移到另一个环境的能力。

常见子特性：

- 适应性。
- 易安装性。
- 可替换性。

游戏样例：

```text
服务端能否从物理机迁移到云服务器；
能否适配不同操作系统、数据库或容器环境。
```

## 使用质量模型

除了产品质量模型，ISO/IEC 25010:2011 还包含使用质量模型。

常见 5 个使用质量特性：

```text
有效性
效率
满意度
免于风险
情境覆盖
```

它关注的是用户在实际使用场景中的效果，而不是软件内部本身。

简单区分：

```text
产品质量：软件本身质量如何。
使用质量：用户用起来效果如何。
```

## 2023 版变化提示

ISO/IEC 25010:2023 的产品质量模型调整为 9 个特性。常见英文名称包括：

```text
Functional suitability
Performance efficiency
Compatibility
Interaction capability
Reliability
Security
Maintainability
Flexibility
Safety
```

与 2011 版相比，比较明显的变化是：

- `Usability` 在新版中更多体现为 `Interaction capability`。
- 新增或突出 `Flexibility`。
- 新增或突出 `Safety`。
- 2011 版的 `Portability` 不再作为同样形式的顶层特性出现。

软考备考时，不建议把 2011 和 2023 混背。选择题通常根据选项判断版本。

## 考试答题模板

如果考试问“ISO/IEC 25010 产品质量模型包括哪些质量特性”，可按常见软考版本回答：

```text
ISO/IEC 25010 软件产品质量模型常见 8 个质量特性：
功能适合性、性能效率、兼容性、易用性、可靠性、安全性、可维护性、可移植性。
其中，功能适合性关注功能满足需求的程度，性能效率关注响应时间和资源利用，
兼容性关注共存和互操作，易用性关注用户理解和操作，可靠性关注持续稳定运行，
安全性关注信息保护，可维护性关注修改和测试的难易程度，可移植性关注跨环境迁移能力。
```
