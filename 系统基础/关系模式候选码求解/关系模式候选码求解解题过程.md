# 关系模式候选码求解解题过程

## 原题

![关系模式候选码求解题目截图](images/关系模式候选码求解-题目截图.png)

## 题目结论

已知关系模式：

```text
R(a, b, c, d)
```

函数依赖集：

```text
F = { a -> cd, c -> b }
```

则 `R` 的候选码是：

```text
a
```

## 核心概念

候选码要同时满足两个条件：

1. 能唯一确定关系模式中的全部属性。
2. 自身不能再删减属性，也就是必须是最小的超码。

判断候选码时，常用方法是求属性闭包。

如果某个属性集 `X` 的闭包 `X+` 包含关系模式 `R` 的全部属性，则 `X` 是超码。

如果 `X` 是超码，并且 `X` 的任意真子集都不是超码，则 `X` 是候选码。

## 第一步：列出所有属性

关系模式 `R(a, b, c, d)` 中的全部属性为：

```text
{a, b, c, d}
```

函数依赖为：

```text
a -> cd
c -> b
```

## 第二步：观察函数依赖右部

函数依赖右部出现的属性有：

```text
a -> cd   右部是 c, d
c -> b    右部是 b
```

所以右部属性集合为：

```text
{b, c, d}
```

属性 `a` 没有出现在任何函数依赖的右部。

考试中可以快速判断：

```text
没有出现在任何函数依赖右部的属性，不能由其他属性推出，因此必须包含在候选码中。
```

因此，候选码中必须包含：

```text
a
```

## 第三步：求 a 的属性闭包

从 `a` 开始求闭包：

```text
a+ = {a}
```

根据函数依赖：

```text
a -> cd
```

可以由 `a` 推出 `c` 和 `d`：

```text
a+ = {a, c, d}
```

又因为现在闭包中已经有 `c`，根据函数依赖：

```text
c -> b
```

可以继续推出 `b`：

```text
a+ = {a, b, c, d}
```

此时 `a+` 已经包含关系模式 `R(a, b, c, d)` 的全部属性。

所以：

```text
a 是超码
```

## 第四步：判断是否最小

候选码要求是最小超码。

这里 `a` 本身只有一个属性，已经不能再删减。

因此：

```text
a 是候选码
```

## 最终答案

```text
R 的候选码是：a
```

## 易错点

1. 不要只看到 `a -> cd` 就以为 `a` 只能推出 `c` 和 `d`，还要继续利用 `c -> b`。
2. 求闭包时要反复应用函数依赖，直到不能推出新属性为止。
3. 没有出现在函数依赖右部的属性通常必须包含在候选码中，本题中就是 `a`。
4. `c` 不能作为候选码，因为 `c+ = {c, b}`，推不出 `a` 和 `d`。
5. `a` 已经能推出全部属性，且无法再删减，所以候选码不是 `ac`、`ad` 等更大的属性集。
