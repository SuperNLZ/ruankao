# 数据库三级模式介绍

## 1. 概述

数据库三级模式结构是数据库系统的经典体系结构，用于描述数据库从用户视图到物理存储的不同抽象层次。

三级模式包括：

```text
外模式 -> 概念模式 -> 内模式
```

它们分别对应用户看到的数据、数据库的全局逻辑结构、数据在存储介质上的物理组织方式。

## 2. 外模式

外模式也称为用户模式或子模式。

它描述某一类用户或某个应用程序所看到和使用的局部数据视图。

特点：

- 面向具体用户或应用。
- 一个数据库可以有多个外模式。
- 不同用户可以看到不同的数据视图。
- 有利于数据安全和访问控制。

例子：

```text
教务系统中，学生看到自己的成绩信息；
教师看到所授课程的学生成绩；
管理员看到完整的学生、课程和成绩数据。
```

## 3. 概念模式

概念模式也称为模式或逻辑模式。

它描述整个数据库的全局逻辑结构，是数据库中全体数据的逻辑表示。

特点：

- 一个数据库只有一个概念模式。
- 描述全局数据结构、属性、联系和约束。
- 不关心数据的具体物理存储方式。
- 是数据库设计中的核心层次。

例子：

```text
学生表、课程表、成绩表，以及它们之间的关系和完整性约束。
```

## 4. 内模式

内模式也称为存储模式。

它描述数据在数据库内部的物理存储结构和存取方式。

特点：

- 一个数据库只有一个内模式。
- 关注数据如何存储在磁盘或存储系统中。
- 包括文件组织、索引、存储路径、数据压缩等。
- 面向数据库管理系统的底层实现。

例子：

```text
某张表使用 B+ 树索引，数据按页存储，索引文件和数据文件分别组织。
```

## 5. 两级映像

为了实现数据独立性，三级模式之间通过两级映像连接。

### 5.1 外模式/概念模式映像

用于描述外模式与概念模式之间的对应关系。

作用：

- 保证逻辑数据独立性。
- 当概念模式发生变化时，可以通过修改映像，使外模式尽量不变。

例子：

```text
数据库中增加了一个字段，但某个用户视图不需要该字段，应用程序可以不受影响。
```

### 5.2 概念模式/内模式映像

用于描述概念模式与内模式之间的对应关系。

作用：

- 保证物理数据独立性。
- 当数据的物理存储方式改变时，可以通过修改映像，使概念模式不变。

例子：

```text
数据库增加索引或调整文件存储结构，但表结构和应用程序不需要改变。
```

## 6. 数据独立性

三级模式和两级映像的核心目的，是提高数据独立性。

### 6.1 逻辑数据独立性

逻辑数据独立性是指：

```text
概念模式发生变化时，外模式和应用程序尽量不受影响。
```

例如：

- 增加属性。
- 增加关系。
- 调整部分逻辑结构。

### 6.2 物理数据独立性

物理数据独立性是指：

```text
内模式发生变化时，概念模式和应用程序尽量不受影响。
```

例如：

- 增加索引。
- 改变文件组织方式。
- 调整存储路径。
- 改变数据存储结构。

## 7. 记忆口诀

```text
外模式面向用户，
概念模式管全局，
内模式管存储；
外概映像保逻辑独立，
概内映像保物理独立。
```

## 8. 常考点

1. 数据库三级模式是：外模式、概念模式、内模式。
2. 一个数据库可以有多个外模式。
3. 一个数据库通常只有一个概念模式和一个内模式。
4. 外模式/概念模式映像保证逻辑数据独立性。
5. 概念模式/内模式映像保证物理数据独立性。
6. 三级模式结构提高了数据库的数据独立性和安全性。
