# 子网掩码、网络号、主机号计算解题过程

## 题目 1

某网络采用定长子网划分，子网掩码为：

```text
255.255.255.248
```

则该网络可支持的主机数是：

```text
6 台
```

## 题目 1 解题过程

### 第一步：把子网掩码换成二进制

```text
255.255.255.248
```

其中：

```text
255 = 11111111
248 = 11111000
```

所以：

```text
255.255.255.248
= 11111111.11111111.11111111.11111000
```

前面连续的 `1` 有：

```text
8 + 8 + 8 + 5 = 29 位
```

因此该子网掩码是：

```text
/29
```

### 第二步：计算主机位数

IPv4 地址一共 32 位。

```text
主机位数 = 32 - 网络位数
        = 32 - 29
        = 3
```

### 第三步：计算可用主机数

3 位主机号可以表示：

```text
2^3 = 8 个地址
```

但每个子网中：

- 主机号全 0 的地址表示网络地址。
- 主机号全 1 的地址表示广播地址。

这两个地址不能分配给普通主机。

所以可用主机数为：

```text
2^3 - 2 = 6
```

因此答案是：

```text
6 台主机
```

## 题目 2

已知：

```text
主机 A IP：192.168.1.130
主机 B IP：192.168.1.200
子网掩码：255.255.255.192
```

问：它们是否属于同一个网络？

答案：

```text
不属于同一个网络
```

## 题目 2 解题过程

### 第一步：分析子网掩码

```text
255.255.255.192
```

其中：

```text
192 = 11000000
```

所以：

```text
255.255.255.192
= 11111111.11111111.11111111.11000000
```

前面连续的 `1` 有：

```text
8 + 8 + 8 + 2 = 26 位
```

因此该子网掩码是：

```text
/26
```

主机位数为：

```text
32 - 26 = 6 位
```

每个子网包含：

```text
2^6 = 64 个地址
```

### 第二步：计算 /26 的网段范围

因为每个子网有 64 个地址，所以最后一段的步长是：

```text
256 - 192 = 64
```

因此 `192.168.1.0/26` 这类网段在最后一段按 64 递增：

```text
192.168.1.0   ~ 192.168.1.63
192.168.1.64  ~ 192.168.1.127
192.168.1.128 ~ 192.168.1.191
192.168.1.192 ~ 192.168.1.255
```

### 第三步：判断主机 A 所在网络

主机 A：

```text
192.168.1.130
```

最后一段 `130` 落在：

```text
128 ~ 191
```

所以主机 A 的网络地址是：

```text
192.168.1.128/26
```

该网段范围：

```text
网络地址：192.168.1.128
可用主机：192.168.1.129 ~ 192.168.1.190
广播地址：192.168.1.191
```

主机 A 的主机号是：

```text
130 - 128 = 2
```

### 第四步：判断主机 B 所在网络

主机 B：

```text
192.168.1.200
```

最后一段 `200` 落在：

```text
192 ~ 255
```

所以主机 B 的网络地址是：

```text
192.168.1.192/26
```

该网段范围：

```text
网络地址：192.168.1.192
可用主机：192.168.1.193 ~ 192.168.1.254
广播地址：192.168.1.255
```

主机 B 的主机号是：

```text
200 - 192 = 8
```

### 第五步：比较网络号

```text
主机 A 网络号：192.168.1.128
主机 B 网络号：192.168.1.192
```

两者网络号不同。

所以：

```text
主机 A 和主机 B 不属于同一个网络
```

## 网络号、主机号、子网掩码的作用

## 1. IP 地址由两部分组成

一个 IPv4 地址可以分为：

```text
网络号 + 主机号
```

例如：

```text
192.168.1.130/26
```

其中：

- 网络号用于标识该 IP 属于哪个网络。
- 主机号用于标识该网络中的哪一台主机。

## 2. 子网掩码的作用

子网掩码用于划分 IP 地址中的网络位和主机位。

规则是：

```text
子网掩码中为 1 的部分表示网络位
子网掩码中为 0 的部分表示主机位
```

例如：

```text
255.255.255.192 = /26
```

表示：

```text
前 26 位是网络位
后 6 位是主机位
```

## 3. 网络号怎么求

网络号的计算方法是：

```text
IP 地址 和 子网掩码 做按位与运算
```

即：

```text
网络号 = IP & 子网掩码
```

考试中，如果只看最后一段，也可以用“步长法”快速计算。

## 4. 主机号怎么求

主机号表示该 IP 在当前子网中的编号。

可以理解为：

```text
主机号 = IP 地址 - 当前子网的网络地址
```

例如：

```text
192.168.1.130/26
网络地址是 192.168.1.128
主机号 = 130 - 128 = 2
```

## 5. 广播地址怎么求

广播地址是当前子网中主机号全为 `1` 的地址。

例如：

```text
192.168.1.128/26
```

该子网有 64 个地址：

```text
192.168.1.128 ~ 192.168.1.191
```

所以：

```text
网络地址：192.168.1.128
广播地址：192.168.1.191
可用主机：192.168.1.129 ~ 192.168.1.190
```

## 快速计算模板

### 模板 1：根据掩码求可用主机数

```text
1. 把子网掩码换成 /n
2. 主机位数 = 32 - n
3. 可用主机数 = 2^(主机位数) - 2
```

例如：

```text
255.255.255.248 = /29
主机位数 = 32 - 29 = 3
可用主机数 = 2^3 - 2 = 6
```

### 模板 2：判断两个 IP 是否同一网络

```text
1. 根据子网掩码确定网络位数或块大小
2. 分别求两个 IP 的网络号
3. 网络号相同，则属于同一网络
4. 网络号不同，则不属于同一网络
```

例如：

```text
192.168.1.130/26 -> 网络号 192.168.1.128
192.168.1.200/26 -> 网络号 192.168.1.192
```

网络号不同，所以不是同一网络。

| 255.255.255.240 | /28 | 16 | 14 |

## 最终答案

```text
题目 1：255.255.255.248 = /29，可用主机数 = 2^3 - 2 = 6。

题目 2：
192.168.1.130/26 属于 192.168.1.128/26。
192.168.1.200/26 属于 192.168.1.192/26。
两者网络号不同，所以不属于同一个网络。
```

## 易错点

1. 可用主机数通常要减去网络地址和广播地址，所以是 `2^主机位 - 2`。
2. `255.255.255.248` 不是 `/28`，而是 `/29`。
3. `255.255.255.192` 是 `/26`，每个子网步长为 `64`。
4. 判断是否同一网络，不是看 IP 前三段是否相同，而是看 `IP & 子网掩码` 后的网络号是否相同。
5. `192.168.1.130` 和 `192.168.1.200` 前三段相同，但在 `/26` 下属于不同子网。
