# Lambda 架构（软考快速记忆）

# 一、什么是 Lambda 架构

entity["scientific_concept","Lambda Architecture","big data processing architecture"] 是一种：

```text
面向大数据的分布式数据处理架构
```

由：

```text
Nathan Marz
```

提出。

核心目标：

```text
同时解决：
大数据量
实时处理
离线计算
```

的问题。

---

# 二、Lambda 架构核心思想

Lambda 架构：

```text
批处理 + 实时处理
```

两套系统共同工作。

目的：

```text
既保证实时性
又保证最终结果准确
```

---

# 三、Lambda 架构三层结构（超高频）

| 层 | 作用 |
|---|---|
| Batch Layer（批处理层） | 全量离线计算 |
| Speed Layer（实时层） | 实时增量处理 |
| Serving Layer（服务层） | 对外提供查询 |

---

# 四、三层详解

# 1. Batch Layer（批处理层）

负责：

```text
全量数据离线处理
```

特点：

- 数据量大
- 延迟高
- 结果准确

常见技术：

- Hadoop
- Spark
- Hive

---

## 工作方式

批处理层：

```text
定期重新计算全部数据
```

例如：

```text
每天凌晨统计：
全站用户行为
```

---

## 优点

```text
结果准确
可重复计算
```

---

## 缺点

```text
实时性差
```

---

# 2. Speed Layer（实时层）

负责：

```text
实时增量计算
```

特点：

- 延迟低
- 数据新鲜
- 结果可能不完全准确

常见技术：

- Storm
- Flink
- Spark Streaming
- Kafka Streams

---

## 工作方式

例如：

```text
用户刚下单
立刻更新销量
```

---

## 优点

```text
实时性强
```

---

## 缺点

```text
结果可能有误差
```

---

# 3. Serving Layer（服务层）

负责：

```text
对外提供查询服务
```

特点：

```text
融合：
批处理结果
+
实时结果
```

常见：

- HBase
- Redis
- Elasticsearch

---

# 五、Lambda 架构工作流程（非常重要）

```text
                原始数据
                    ↓
        ┌───────────────┐
        ↓               ↓
  Batch Layer      Speed Layer
   全量离线           实时增量
        ↓               ↓
        └────→ Serving Layer
                    ↓
                 用户查询
```

---

# 六、为什么需要 Lambda 架构

因为：

```text
只有离线处理：
实时性不够
```

例如：

```text
每天统计一次销量
```

用户刚下单：

```text
看不到最新结果
```

---

而：

```text
只有实时处理：
准确性可能不足
```

因此：

```text
离线 + 实时
共同工作
```

---

# 七、Lambda 架构优点（高频）

## 1. 同时支持：

```text
大数据
实时计算
```

---

## 2. 容错性强

因为：

```text
原始数据永久保存
```

可以：

```text
重新计算
```

---

## 3. 查询性能高

Serving Layer：

```text
提前生成结果
```

---

## 4. 最终一致性

实时层先给近似结果：

随后：

```text
批处理层纠正
```

---

# 八、Lambda 架构缺点（超高频）

## 最大缺点：

```text
系统复杂
```

因为：

```text
同一逻辑
写两套
```

即：

- 批处理逻辑
- 实时处理逻辑

维护成本高。

---

# 九、Lambda vs Kappa（高频）

| 对比 | Lambda | Kappa |
|---|---|---|
| 批处理层 | 有 | 无 |
| 实时层 | 有 | 有 |
| 架构复杂度 | 高 | 低 |
| 数据处理 | 双路径 | 单路径 |

---

# 十二、高频易错点

| 易错点 | 正确理解 |
|---|---|
| Lambda 只有实时层 | 错，还有批处理层 |
| Lambda 不支持实时 | 错，实时层是核心组成 |
| Batch Layer 实时处理 | 错，负责离线全量计算 |
| Speed Layer 最终最准确 | 错，最终准确由批处理层保证 |

---

# 十三、典型应用场景

Lambda 架构常用于：

- 实时推荐
- 日志分析
- 电商统计
- 风控系统
- 用户行为分析
- 大数据平台

---

# 十四、一句话总结

> Lambda 架构通过“批处理层 + 实时层 + 服务层”三层结构，实现了大数据场景下实时性与准确性的平衡。
