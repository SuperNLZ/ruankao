# 软件模块扇入与扇出（软考快速记忆）

## 题目

一个模块直接调用下层模块的数量用（）表示。

A. 扇入  
B. 扇出  
C. 宽度  
D. 广度

---

# 正确答案

## B：扇出

---

# 一、什么是扇出（Fan-Out）

扇出表示：

```text
一个模块直接调用的下层模块数量
```

即：

```text
“我调用了多少别人”
```

---

# 二、什么是扇入（Fan-In）

扇入表示：

```text
有多少上层模块调用当前模块
```

即：

```text
“多少别人调用我”
```

---

# 三、图形理解（非常重要）

```text
        A
      / | \
     B  C  D
```

对于模块 A：

```text
扇出 = 3
```

因为：

```text
A 调用了 B、C、D
```

---

对于模块 B：

```text
扇入 = 1
```

因为：

```text
只有 A 调用了 B
```

---

# 四、考试核心记忆

## 扇出（Fan-Out）

```text
向外调用别人
```

关键词：

```text
调用下层模块
```

---

## 扇入（Fan-In）

```text
别人向内调用我
```

关键词：

```text
被多少模块调用
```

---

# 五、为什么选 B

题目：

```text
一个模块直接调用下层模块的数量
```

这正是：

```text
扇出
```

因此答案：

```text
B
```

---

# 六、软考高频考点

## 高扇入

表示：

```text
模块复用程度高
```

通常较好。

---

## 高扇出

表示：

```text
模块依赖太多模块
```

耦合度高。

通常不太好。

---

# 七、快速记忆口诀

## 扇出

```text
我调用别人
向外散开
```

---

## 扇入

```text
别人调用我
向内汇聚
```

---

# 八、易错点

| 题目描述 | 正确概念 |
|---|---|
| 调用下层模块数量 | 扇出 |
| 被多少模块调用 | 扇入 |
| 向外扩散 | 扇出 |
| 向内汇聚 | 扇入 |

---

# 一句话总结

> 扇出表示模块调用下层模块的数量；扇入表示调用该模块的上层模块数量。
