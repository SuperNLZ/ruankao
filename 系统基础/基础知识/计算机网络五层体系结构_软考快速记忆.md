# 计算机网络五层体系结构（软考快速记忆）

# 一、什么是五层体系结构

计算机网络中最常用的教学模型：

```text
五层体系结构
```

是在：

- OSI 七层模型
- TCP/IP 四层模型

基础上简化得到的。

---

# 二、五层结构（超高频）

从上到下：

| 层次 | 核心作用 | 常见协议 |
|---|---|---|
| 应用层 | 用户应用服务 | HTTP、DNS、FTP |
| 传输层 | 端到端通信 | TCP、UDP |
| 网络层 | 路由与寻址 | IP、ICMP |
| 数据链路层 | 相邻节点通信 | Ethernet、MAC |
| 物理层 | 比特传输 | 网线、光纤 |

---

# 三、协议所属层（考试必考）

| 协议 | 所属层 |
|---|---|
| HTTP | 应用层 |
| HTTPS | 应用层 |
| DNS | 应用层 |
| TCP | 传输层 |
| UDP | 传输层 |
| IP | 网络层 |
| ICMP | 网络层 |
| Ethernet | 数据链路层 |

---

# 四、各层详细说明

# 1. 应用层（Application Layer）

最接近用户。

负责：

```text
应用程序之间通信
```

例如：

- 浏览网页
- 发邮件
- 文件传输

---

## 常见协议

| 协议 | 作用 |
|---|---|
| HTTP | 网页访问 |
| HTTPS | 安全网页 |
| FTP | 文件传输 |
| DNS | 域名解析 |
| SMTP | 邮件发送 |

---

# 2. 传输层（Transport Layer）

负责：

```text
端到端通信
```

核心：

```text
可靠性
```

---

## TCP

特点：

```text
可靠
面向连接
```

支持：

- 重传
- 排序
- 流量控制

适用于：

- 网页
- 支付
- 文件传输

---

## UDP

特点：

```text
不可靠
但速度快
```

适用于：

- 视频直播
- 游戏
- 语音通话

---

# 3. 网络层（Network Layer）

负责：

```text
跨网络寻址与路由
```

核心：

```text
IP地址
```

例如：

```text
192.168.1.1
```

---

## 常见协议

| 协议 | 作用 |
|---|---|
| IP | 网络寻址 |
| ICMP | 差错控制 |
| ARP | 地址解析 |

---

## 路由器工作层次

```text
网络层
```

---

# 4. 数据链路层（Data Link Layer）

负责：

```text
同一局域网通信
```

核心：

```text
MAC地址
```

例如：

```text
00-1A-2B-3C
```

---

## 核心功能

- 封装成帧
- 差错检测
- 透明传输

---

## 交换机工作层次

```text
数据链路层
```

---

# 5. 物理层（Physical Layer）

最底层。

负责：

```text
真正传输0和1
```

例如：

- 网线
- 光纤
- 无线电波
- 电信号

---

# 五、数据封装过程（超高频）

发送时：

```text
HTTP数据
↓
TCP段
↓
IP包
↓
MAC帧
↓
比特流
```

接收时反向拆包。

---

# 九、高频易错点

| 易错点 | 正确理解 |
|---|---|
| HTTP 属于传输层 | 错，属于应用层 |
| IP 属于传输层 | 错，属于网络层 |
| UDP 属于网络层 | 错，属于传输层 |
| 路由器工作在链路层 | 错，工作在网络层 |
| 交换机负责路由 | 错，交换机主要基于MAC地址 |
