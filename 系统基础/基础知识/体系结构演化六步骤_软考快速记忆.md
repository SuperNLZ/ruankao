# 体系结构演化六步骤（软考标准版）

# 一、体系结构演化六步骤（超高频）

```text
1. 需求变化归类
2. 制订体系结构演化计划
3. 构件变动
4. 更新构件的相互作用
5. 构件组装与测试
6. 技术评审
```

最终得到：

```text
演化后的体系结构
```

---

# 二、六步骤核心含义

| 步骤 | 核心内容 |
|---|---|
| 需求变化归类 | 分析变化属于什么类型 |
| 制订体系结构演化计划 | 规划如何修改架构 |
| 构件变动 | 修改系统构件 |
| 更新构件的相互作用 | 修改构件间连接关系 |
| 构件组装与测试 | 集成并测试系统 |
| 技术评审 | 评估演化结果是否合理 |

---

# 三、核心思想（高频）

体系结构演化不仅修改：

```text
构件（Component）
```

还需要修改：

```text
构件之间的交互关系（Connector）
```

这是：

```text
软考高频考点
```

---

```text
需求变化

制定计划

修改交互

组装测试

演化后的体系结构
```

# 六、高频易错点

| 易错点 | 正确理解 |
|---|---|
| 体系结构演化只修改代码 | 错，还包括交互关系 |
| 不需要修改连接件 | 错，连接件也需要演化 |
| 技术评审可省略 | 错，是标准步骤 |
| 这是普通开发流程 | 错，这是体系结构演化流程 |

---

# 七、一句话总结

> 体系结构演化包括需求变化归类、制订体系结构演化计划、构件变动、更新构件的相互作用、构件组装与测试以及技术评审，最终形成演化后的体系结构。
