# 效用树（Utility Tree）（软考快速记忆）

# 一、什么是效用树

效用树（Utility Tree）是：

```text
ATAM 中用于组织和分析质量属性场景
```

的一种树形结构。

主要用于：

```text
软件架构评估
```

核心作用：

```text
把抽象的质量需求
转换成具体可分析的场景
```

---

# 二、效用树属于哪里（超高频）

效用树：

```text
属于 ATAM（架构权衡分析方法）
```

ATAM：

```text
Architecture Tradeoff Analysis Method
架构权衡分析方法
```

用于：

```text
分析软件架构是否满足质量属性需求
```

---

# 三、为什么需要效用树

因为：

```text
质量需求通常很抽象
```

例如：

- 系统要高性能
- 系统要安全
- 系统要易扩展

这些：

```text
无法直接评估
```

因此需要：

```text
拆解成具体场景
```

这就是：

```text
效用树
```

---

# 四、效用树结构（超高频）

效用树通常：

```text
树根
  ↓
质量属性
  ↓
属性分类
  ↓
质量属性场景（叶子节点）
```

---

# 五、结构图（非常重要）

```text
                    效用树
                       │
 ┌──────────┬──────────┬──────────┐
 性能       安全性      可修改性
  │           │            │
响应时间    身份认证      模块扩展
  │           │            │
10000并发   非法访问      新增支付方式
2秒内响应   必须拦截      1周内完成
```

---

# 六、效用树中的“场景”（超高频）

效用树最底层：

```text
质量属性场景
```

即：

```text
系统在某种条件下
应该如何响应
```

---

# 七、场景六要素（超高频）

ATAM 场景通常包括：

| 要素 | 含义 |
|---|---|
| 刺激源（Source） | 谁触发事件 |
| 刺激（Stimulus） | 发生什么事 |
| 环境（Environment） | 在什么环境下 |
| 制品（Artifact） | 影响哪个部分 |
| 响应（Response） | 系统如何响应 |
| 响应度量（Response Measure） | 如何衡量效果 |

---

# 八、效用树如何排序（超高频）

ATAM 中：

```text
场景需要排序
```

排序依据：

| 维度 | 含义 |
|---|---|
| 重要性 | 对系统成功的重要程度 |
| 风险/难度 | 实现难度或技术风险 |

---

# 九、ATAM 最关注什么

重点关注：

```text
高重要
高风险
```

的场景。

因为：

```text
最容易暴露架构问题
```

---

# 十、效用树构建流程（高频）

## 第一步

确定：

```text
质量属性
```

例如：

- 性能
- 安全性
- 可用性

---

## 第二步

继续细化：

```text
属性分类
```

例如：

性能：

- 响应时间
- 吞吐量

---

## 第三步

提出：

```text
质量属性场景
```

例如：

```text
10000用户并发时
系统2秒内响应
```

---

## 第四步

评估：

```text
重要性 + 风险
```

---

# 十三、效用树中的常见概念（超高频）

| 概念 | 含义 |
|---|---|
| 敏感点 | 对质量属性影响大的设计点 |
| 权衡点 | 一个设计影响多个质量属性 |
| 风险点 | 可能导致问题的设计 |
| 非风险点 | 已验证合理的设计 |

---

# 十四、经典真题（软考高频）

## 题目

在软件架构评估中，（1）方法采用效用树这一工具来对质量属性进行分类和优先级排序。效用树的结构包括：（2）。

### 问题1

A. SAEM  
B. ATAM  
C. SAAM  
D. CBAM

### 问题2

A. 树根 → 质量属性 → 属性分类 → 质量属性场景（叶子节点）  
B. 树根 → 属性分类 → 属性描述 → 质量属性场景（叶子节点）  
C. 树根 → 质量属性 → 属性描述 → 质量属性场景（叶子节点）  
D. 树根 → 功能需求 → 需求描述 → 质量属性场景（叶子节点）

---

# 十五、答案

## 问题1

```text
B：ATAM
```

因为：

```text
效用树是 ATAM 的核心工具
```

用于：

```text
质量属性分析与优先级排序
```

---

## 问题2

```text
A：树根 → 质量属性 → 属性分类 → 质量属性场景（叶子节点）
```

这是：

```text
效用树标准结构
```

---

# 十六、为什么其他选项错

## 问题1

### A SAEM

不是软考标准架构评估主流方法。

---

### C SAAM

SAAM：

```text
Software Architecture Analysis Method
```

主要用于：

```text
可修改性分析
```

而：

```text
效用树属于 ATAM
```

---

### D CBAM

CBAM：

```text
Cost Benefit Analysis Method
```

强调：

```text
成本收益分析
```

不是效用树核心方法。

---

## 问题2

### A 对

```text
树根 → 质量属性 → 属性分类 → 质量属性场景
```

符合效用树常见结构。

---

### B 错

缺少：

```text
质量属性
```

---

### C 错

```text
属性描述
```

不是该题参考答案采用的标准层次。

---

### D 错

效用树：

```text
分析质量属性
```

不是功能需求树。

# 十九、高频易错点

| 易错点 | 正确理解 |
|---|---|
| 效用树只关注性能 | 错，还包括安全、可用性等 |
| 效用树输出代码结构 | 错，输出质量属性场景 |
| 场景不需要量化 | 错，通常需要明确度量 |
| ATAM 不分析风险 | 错，风险分析是核心 |
| 效用树结构是“属性描述” | 错，本题标准答案是“属性分类” |
