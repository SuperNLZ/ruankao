# 模块化程序设计中的内聚-快速记忆

## 一句话

```text
内聚 = 模块内部各部分联系有多紧密。
```

考试目标：

```text
高内聚，低耦合。
```

## 强弱顺序

从弱到强：

```text
偶然 < 逻辑 < 时间 < 过程 < 通信 < 顺序 < 功能
```

口诀：

```text
偶逻时过通顺功
```

记住：

```text
偶然最差，功能最好。
```

## 7 个关键词

| 内聚类型 | 看到这些词就选 |
|---|---|
| 偶然内聚 | 没关系、凑一起 |
| 逻辑内聚 | 同一类、按条件选 |
| 时间内聚 | 同一时间、初始化、结束 |
| 过程内聚 | 按步骤、按流程 |
| 通信内聚 | 同一数据 |
| 顺序内聚 | 前一步输出给后一步 |
| 功能内聚 | 单一功能、明确职责 |

## 最容易混的 3 组

### 过程 vs 顺序

```text
过程：只强调步骤顺序。
顺序：前一步输出是后一步输入。
```

### 通信 vs 顺序

```text
通信：大家用同一份数据。
顺序：数据从前一步流到后一步。
```

### 逻辑 vs 功能

```text
逻辑：一类功能放一起，每次选一种。
功能：所有内容只为一个明确功能服务。
```

## 最终背诵版

```text
内聚表示模块内部各部分联系的紧密程度。
内聚从弱到强：偶然、逻辑、时间、过程、通信、顺序、功能。
口诀：偶逻时过通顺功。
偶然内聚最弱，功能内聚最强。
```
