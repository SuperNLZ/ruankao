# 模块化程序设计中的内聚-概念讲解

## 什么是内聚

内聚是模块化程序设计中的一个重要概念，用来衡量一个模块内部各个组成部分之间联系的紧密程度。

简单说：

```text
内聚描述的是：模块内部的代码、数据和功能是不是围绕同一个明确目标组织在一起。
```

内聚性越高，说明模块内部越集中、越专一、越容易理解和维护。

## 内聚和模块质量

在模块化设计中，通常追求：

```text
高内聚，低耦合
```

其中：

- 高内聚：模块内部职责集中。
- 低耦合：模块之间依赖较少。

一个好的模块应该尽量只完成一个明确的功能，模块内部的代码都服务于这个功能。

## 为什么要提高内聚

高内聚的模块有以下优点：

1. 容易理解。
2. 容易测试。
3. 容易维护。
4. 容易复用。
5. 修改影响范围小。

例如，一个“玩家背包模块”如果只负责背包物品的增加、删除、查询和整理，那么它的职责就比较清晰，内聚性较高。

如果这个模块同时还负责登录、支付、活动配置、排行榜计算，那么它的职责就混杂，内聚性较低。

## 内聚的类型

内聚通常按强弱程度分为以下几类：

```text
偶然内聚
逻辑内聚
时间内聚
过程内聚
通信内聚
顺序内聚
功能内聚
```

它们的内聚强度从低到高排列为：

```text
偶然内聚 < 逻辑内聚 < 时间内聚 < 过程内聚 < 通信内聚 < 顺序内聚 < 功能内聚
```

## 1. 偶然内聚

偶然内聚是最弱的内聚。

它指模块中的各个成分之间没有明显关系，只是偶然被放在同一个模块中。

示例：

```text
一个工具模块中同时包含：
登录校验
字符串格式化
邮件发送
排行榜排序
数据库备份
```

这些功能之间缺乏明确联系，只是被随意放在一起，因此属于偶然内聚。

## 2. 逻辑内聚

逻辑内聚是指模块中的各个功能在逻辑上属于同一类，但每次调用时只执行其中某一种功能。

示例：

```text
一个输入处理模块中包含：
键盘输入处理
鼠标输入处理
手柄输入处理
触屏输入处理
```

这些功能都属于“输入处理”，但具体执行哪一种由参数或条件决定。

逻辑上相关，但模块内部功能仍然偏杂。

## 3. 时间内聚

时间内聚是指模块中的多个操作因为在同一时间段执行而被放在一起。

示例：

```text
系统初始化模块中包含：
读取配置
初始化日志
连接数据库
加载缓存
启动定时任务
```

这些操作都在系统启动时执行，因此形成时间内聚。

它们的共同点是“发生时间相同”，但功能目标不一定完全一致。

## 4. 过程内聚

过程内聚是指模块中的多个操作按照特定的执行顺序组织在一起。

示例：

```text
购买流程模块：
校验参数
检查余额
扣除货币
发放道具
返回结果
```

这些步骤按照业务流程顺序执行，因此属于过程内聚。

过程内聚比时间内聚更强，因为它不仅有时间关系，还有执行顺序关系。

## 5. 通信内聚

通信内聚是指模块中的多个操作都使用同一组输入数据，或产生同一组输出数据。

示例：

```text
玩家信息处理模块：
读取玩家基础信息
计算玩家战力
刷新玩家展示信息
保存玩家状态
```

这些操作都围绕“玩家信息”这一组数据进行，因此属于通信内聚。

通信内聚强调：

```text
围绕同一数据工作
```

## 6. 顺序内聚

顺序内聚是指模块中一个处理步骤的输出，作为下一个处理步骤的输入。

示例：

```text
订单处理模块：
生成订单 -> 支付校验 -> 发放道具 -> 生成流水
```

前一步的结果会影响后一步，步骤之间有输入输出衔接关系，因此属于顺序内聚。

顺序内聚比通信内聚更强，因为它不仅共享数据，还存在明显的数据流动关系。

## 7. 功能内聚

功能内聚是最强、最理想的内聚。

它指模块中的所有成分都为完成同一个明确功能服务。

示例：

```text
登录认证模块：
接收登录请求
校验账号密码
生成登录令牌
返回登录结果
```

这些操作都围绕“完成登录认证”这一单一功能展开，因此属于功能内聚。

功能内聚的模块职责最清晰，也最容易维护和复用。

## 手游开发样例

以游戏商城系统为例。

### 低内聚设计

```text
商城模块同时负责：
商品购买
玩家登录
活动配置
邮件发送
排行榜结算
数据库备份
```

这些职责之间关系不紧密，模块边界混乱，属于低内聚设计。

### 高内聚设计

可以拆分为多个职责清晰的模块：

```text
账号模块：负责登录、鉴权、令牌管理
商城模块：负责商品查询、购买、订单处理
背包模块：负责道具发放、道具查询、道具消耗
活动模块：负责活动配置、活动奖励、活动状态
排行榜模块：负责排行计算、排行查询
```

这样每个模块只围绕一个明确职责工作，内聚性更高。

## 考试答题模板

如果考试问“什么是内聚，内聚有哪些类型”，可以这样回答：

```text
内聚是指一个模块内部各组成部分之间联系的紧密程度，是衡量模块独立性的重要指标。
内聚性越高，模块职责越集中，越容易理解、维护、测试和复用。
模块化设计通常追求高内聚、低耦合。

常见内聚类型按强度从低到高依次为：
偶然内聚、逻辑内聚、时间内聚、过程内聚、通信内聚、顺序内聚、功能内聚。
其中偶然内聚最弱，功能内聚最强。
```

## 总结

内聚关注的是模块内部关系。

```text
内聚越高，模块越专一；
内聚越低，模块越混杂。
```

模块化设计应尽量避免偶然内聚、逻辑内聚等低内聚形式，追求功能内聚。
