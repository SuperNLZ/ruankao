# 软件开发模型与风险评估-快速记忆

## 一句话

```text
看到“风险评估/风险分析/风险控制”，选螺旋模型。
```

## 四个模型关键词

| 模型 | 关键词 |
|---|---|
| 瀑布模型 | 线性、阶段顺序、需求稳定 |
| 螺旋模型 | 风险分析、风险评估、风险控制、迭代 |
| V 模型 | 验证、确认、测试对应 |
| 增量模型 | 分批交付、逐步增加功能 |
| W 模型 | 整个软件开发周期|

## 最短口诀

```text
风险找螺旋
线性找瀑布
测试找 V
分批找增量
```

## 选择题判断法

```text
风险评估 -> 螺旋模型
严格阶段 -> 瀑布模型
验证确认 -> V 模型
逐步交付 -> 增量模型
```

## 易错点

```text
V 模型强调测试，不强调风险。
增量模型强调交付，不强调风险。
瀑布模型强调顺序，不强调风险。
螺旋模型才强调风险。
```

## 最终背诵版

```text
螺旋模型结合迭代和风险分析，每个开发周期都包含风险评估环节，
适合大型、复杂、高风险项目。
看到风险评估，优先选螺旋模型。
W 模型强调测试伴随着整个软件开发周期，而且测试的对象不仅仅是程序，需求、设计等同样要测试，也就是说，测试与开发是同步进行的。
```
