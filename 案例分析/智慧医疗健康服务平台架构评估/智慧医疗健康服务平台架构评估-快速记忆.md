# 智慧医疗健康服务平台架构评估-快速记忆

## 0. 题目原文

试题一，25 分。

阅读以下关于软件架构评估的说明，回答下列问题。

说明：

某市卫健委计划构建一个市级“智慧医疗健康服务平台”，旨在整合全市三甲医院、社区诊所及医保系统的资源，为市民提供统一的预约挂号、在线问诊、电子病历查阅和医保结算服务。在需求分析阶段，提炼出以下关键质量属性场景：

```text
(a) 医保结算功能模块需支持未来省级医保平台政策的变更，并在 3 人月内完成适配开发；
(b) 系统需保证 7x24 小时不间断服务，全年正式运行时间不低于 99.99%；
(c) 门诊检查报告不可篡改；
(d) 门诊流量高峰期间（如工作日早 8 点至 10 点），预约挂号页面的响应时间应小于 1 秒；
(e) 患者的电子病历数据在存储和传输过程中必须进行加密和脱敏处理，防止隐私泄露；
(f) 开发团队在为平台新增“疫苗接种记录查询”功能时，应最多在 2 周内完成并与现有系统集成；
(g) 若主数据库节点发生故障，应在 90 秒内无缝切换至备用数据库，且不丢失任何已提交事务；
(h) 平台为第三方提供标准 API，每天至少支持 10000 次调用。
```

在此基础上开发部门目前正在组织系统开发的相关人员对系统架构进行评估。

问题 1，12 分：

在架构评估过程中，质量属性效用树（utility tree）是对系统质量属性进行识别和优先级排序的重要工具。请给出合适的质量属性，填入图 1 中 (1)、(2) 空白处；并选择题干描述中的 (a)～(h)，将恰当的序号填入 (3)～(6) 空白处，完成该系统的效用树。

问题 2，4 分：

质量属性场景 (h) 同时还关注另一个运行期质量属性，请指出它并予以简单解释。

问题 3，9 分：

患者的电子病历数据在传输过程中必须进行加密和脱敏处理，张工提出采用管道-过滤器风格，李工提议采用虚拟机风格。请选择一种更适合的架构风格，并以 300 字以内的文字从性能、可维护性、安全性方面说明理由。

## 2. 问题 1 答案

填空结论：

| 空白 | 答案 |
|---|---|
| (1) | 可修改性 |
| (2) | 性能 |
| (3) | (a) |
| (4) | (g) |
| (5) | (e) |
| (6) | (d) |

完整效用树：

| 质量属性 | 对应场景 |
|---|---|
| 可修改性 | (a) 医保政策变更 3 人月内适配；(f) 新增疫苗接种查询 2 周内集成 |
| 可用性 | (b) 7x24 小时且 99.99%；(g) 主库故障 90 秒内无缝切换且不丢事务 |
| 安全性 | (c) 报告不可篡改；(e) 病历数据加密和脱敏 |
| 性能 | (d) 高峰期预约挂号响应小于 1 秒；(h) 标准 API 每天至少 10000 次调用 |

判断方法：

```text
改功能、适配政策、加新功能 -> 可修改性
不断服务、故障切换、恢复时间 -> 可用性
不可篡改、加密、脱敏、防泄露 -> 安全性
响应时间、吞吐量、并发量、调用次数 -> 性能
```

## 3. 问题 2 答案

质量属性场景 (h) 同时关注：

```text
互操作性。
```

解释：

```text
(h) 中“每天至少支持 10000 次调用”体现性能中的吞吐量要求；
“为第三方提供标准 API”则体现互操作性，表示平台能够通过统一接口与外部第三方系统进行数据交换、功能调用和系统集成。
```

## 4. 问题 3 答案

更适合的架构风格：

```text
管道-过滤器风格。
```

300 字以内答题模板：

```text
应采用管道-过滤器风格。电子病历传输过程中的校验、脱敏、加密、审计等处理可以拆成独立过滤器，数据按管道顺序流动。性能上，该风格适合流式处理，可减少整体阻塞，并支持部分过滤器并行或替换优化；可维护性上，各过滤器职责单一，脱敏规则或加密算法变化时只需修改对应过滤器；安全性上，可把隐私处理、安全校验和加密逻辑集中在固定环节，便于审计和复用。虚拟机风格更适合解释执行中间代码或脚本，额外开销较大，不是该场景首选。
```

## 5. 核心速记

```text
效用树四类：改、用、安、性。

改：a 政策适配，f 新功能集成。
用：b 99.99%，g 故障切换。
安：c 不可篡改，e 加密脱敏。
性：d 1 秒响应，h 10000 次调用。

h：性能 + 互操作性。

病历传输加密脱敏：选管道-过滤器，不选虚拟机。
```

## 6. 易错点

1. (a) 和 (f) 不要放到性能，它们强调变化成本和集成时间，应归入可修改性。
2. (g) 不只是数据库技术题，90 秒切换且不丢事务是典型可用性场景。
3. (h) 在效用树里可按“10000 次调用”归入性能，但“标准 API”还体现互操作性。
4. 管道-过滤器适合数据处理流水线；虚拟机风格适合解释执行和跨平台运行环境，不是电子病历传输加密脱敏的首选。
