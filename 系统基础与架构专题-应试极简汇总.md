# 系统基础与架构专题-应试极简汇总

> 只用于考前速记：不讲原理，只背关键词、公式、结论。

## 计算题

| 题型 | 极简记忆 |
|---|---|
| 地址范围容量 | `容量 = 末地址 - 首地址 + 1` |
| A0000H~CFFFFH | `CFFFF-A0000+1 = 30000H = 192KB` |
| 64K*8bit 芯片 | `8bit=1B`，每片 `64KB` |
| 192KB 用 64KB 芯片 | `192/64=3` 片 |
| /29 主机数 | `32-29=3`，`2^3-2=6` |
| 255.255.255.248 | `/29`，每子网 8 地址，可用 6 主机 |
| 255.255.255.192 | `/26`，步长 `64`，可用 `62` 主机 |
| 192.168.1.130/26 | 属于 `192.168.1.128/26` |
| 192.168.1.200/26 | 属于 `192.168.1.192/26` |
| 判断同网 | 比较 `IP & 掩码` 后的网络号 |

## 文件索引节点

| 关键词 | 极简记忆 |
|---|---|
| 地址项数 | `索引块大小 / 地址项大小` |
| 1KB 索引块，4B 地址项 | `1024/4=256` 个地址 |
| 5 个直接索引 | 逻辑块 `0~4` |
| 第 1 个一级间接 | 逻辑块 `5~260` |
| 第 2 个一级间接 | 逻辑块 `261~516` |
| 二级间接 | 指向“一级索引块地址表”，不是数据块 |
| 本题逻辑块 5 | 物理块 `58` |
| 本题逻辑块 261 | 物理块 `187` |
| 本题 101 号物理块 | 二级间接地址索引表 |

## 数据库三级模式

| 概念 | 极简记忆 |
|---|---|
| 三级模式 | `外模式 -> 概念模式 -> 内模式` |
| 外模式 | 用户视图；可有多个 |
| 概念模式 | 全局逻辑结构；通常一个 |
| 内模式 | 物理存储结构；通常一个 |
| 外/概映像 | 保证逻辑数据独立性 |
| 概/内映像 | 保证物理数据独立性 |

口诀：

```text
外看用户，概管全局，内管存储；
外概保逻辑，概内保物理。
```

## 白盒测试

| 关键词 | 选什么 |
|---|---|
| 看内部代码、路径、分支 | 白盒测试 |
| 看需求、输入输出、功能 | 黑盒测试 |
| 每条语句至少一次 | 语句覆盖 |
| 判定真假分支都走 | 判定覆盖/分支覆盖 |
| 每个条件真假都出现 | 条件覆盖 |
| 判定真假 + 条件真假 | 判定/条件覆盖 |
| 条件所有取值组合 | 条件组合覆盖 |
| 所有执行路径 | 路径覆盖 |

覆盖强度简记：

```text
语句 < 判定/条件 < 条件组合
路径最强但难穷尽
```

## 内聚

| 内容 | 极简记忆 |
|---|---|
| 内聚 | 模块内部联系紧密程度 |
| 设计目标 | 高内聚，低耦合 |
| 从弱到强 | 偶然 < 逻辑 < 时间 < 过程 < 通信 < 顺序 < 功能 |
| 口诀 | `偶逻时过通顺功` |

| 关键词 | 选什么 |
|---|---|
| 没关系、凑一起 | 偶然内聚 |
| 同一类、按条件选 | 逻辑内聚 |
| 同一时间、初始化、结束 | 时间内聚 |
| 按步骤、按流程 | 过程内聚 |
| 同一数据 | 通信内聚 |
| 前一步输出给后一步 | 顺序内聚 |
| 单一功能、明确职责 | 功能内聚 |

## ISO/IEC 25010

| 内容 | 极简记忆 |
|---|---|
| 常考版本 | 2011 版产品质量模型 |
| 八大特性口诀 | `功性兼用，可安维移` |

| 特性 | 关键词 |
|---|---|
| 功能适合性 | 功能完整、正确、满足需求 |
| 性能效率 | 响应时间、资源、容量、并发 |
| 兼容性 | 共存、互操作 |
| 易用性 | 易学、易操作、界面、防误操作 |
| 可靠性 | 可用、容错、恢复、稳定 |
| 安全性 | 保密、完整、防抵赖、认证、审计 |
| 可维护性 | 模块化、易修改、易测试、易分析 |
| 可移植性 | 适配、安装、迁移、替换 |

易混：

```text
可靠性：别挂，挂了能恢复。
安全性：别泄露，别伪造。
兼容性：和别人一起工作。
可移植性：自己换环境运行。
```

```text

客户端不直接访问数据库。
```

## SOA

| 内容 | 极简记忆 |
|---|---|
| SOA | 面向服务架构 |
| 核心 | 把业务能力封装成服务供多个系统复用 |
| 三角色 | 服务提供者、服务消费者、服务注册中心 |
| 常见关键词 | 服务、松耦合、标准接口、复用、注册发现、组合、ESB |

口诀：

```text
服务封装，松耦复用；注册发现，组合治理。
```

SOA vs 微服务：

```text
SOA：偏企业系统集成，粒度较粗，常见 ESB。
微服务：偏系统内部拆分，粒度较细，独立部署。
```

## 断路器模式

| 状态 | 极简记忆 |
|---|---|
| Closed | 正常调用 |
| Open | 快速失败，不再调用 |
| Half-Open | 少量请求试探恢复 |

口诀：

```text
关着能通，开了拒绝，半开试探。
```

易混：

```text
重试：失败后再试。
断路器：失败太多后先别试。
限流：请求太多不让进。
降级：返回简化/备用结果。
```

## Kruchten 4+1 视图

| 视图 | 关键词 |
|---|---|
| 逻辑视图 | 功能、类、对象、业务逻辑 |
| 开发视图 | 模块、包、组件、代码组织、分层 |
| 进程视图 | 进程、线程、并发、通信、性能 |
| 物理视图 | 服务器、节点、网络、部署、拓扑 |
| 场景视图 | 用例、场景、关键流程、验证架构 |

口诀：

```text
逻辑看功能，开发看代码，进程看运行，物理看部署，场景看用例验证。
```
