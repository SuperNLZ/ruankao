# 523案例结构图-考前速记

## 1. 总体做题框架

案例分析不是写作文，核心是识别题型并套关键词。

```mermaid
flowchart TD
    A["读题干"] --> B["圈场景：系统是什么"]
    B --> C["圈问题：性能 / 可用 / 安全 / 数据 / 架构"]
    C --> D["判断题型"]
    D --> D1["新技术架构题"]
    D --> D2["数据缓存题"]
    D --> D3["质量属性评估题"]
    D --> D4["DFD/UML 建模题"]
    D --> D5["嵌入式/边缘计算题"]
    D1 --> E["按模板列架构组成 + 流程 + 风险治理"]
    D2 --> F["按读写流程 + 异常问题 + 一致性处理"]
    D3 --> G["按质量属性 + 场景六要素 + 风险敏感权衡"]
    D4 --> H["按图元素和上下文平衡检查"]
    D5 --> I["按分层架构 + 实时性可靠性安全性"]
```

## 2. 第一优先级：大模型 / RAG / 知识图谱

### 2.1 结构图

```mermaid
flowchart TD
    A["用户提问"] --> B["认证与权限识别"]
    B --> C["问题改写 / 意图识别 / 关键词提取"]
    C --> D["混合检索"]
    D --> D1["向量数据库：语义相似检索"]
    D --> D2["关键词索引：错误码、接口名、活动编号"]
    D --> D3["知识图谱：实体关系和路径推理"]
    D1 --> E["结果去重与重排"]
    D2 --> E
    D3 --> E
    E --> F["权限过滤和时效过滤"]
    F --> G["Prompt 拼接：片段 + 来源 + 约束"]
    G --> H["大模型生成答案"]
    H --> I["返回答案、引用来源、置信提示"]
    I --> J["记录日志和用户反馈"]
```

### 2.2 答题口令

```text
先治理知识，再混合检索，最后让大模型基于证据回答。
```

### 2.3 必写 5 点

```text
1. 文档解析、清洗、切分、元数据标注、向量化。
2. 向量库做语义检索，关键词索引做精确检索，知识图谱做关系推理。
3. RAG 流程包括认证、问题改写、混合检索、重排、Prompt 拼接、大模型生成。
4. 防幻觉靠引用来源、规则校验、知识图谱校验、资料不足时拒答。
5. 防泄露靠权限过滤、脱敏加密、模型网关和审计日志。
```

## 3. 第二优先级：云原生微服务 / 事件驱动

### 3.1 结构图

```mermaid
flowchart TD
    A["单体或集中式系统问题"] --> B["按业务边界拆服务"]
    B --> B1["账号服务"]
    B --> B2["支付服务"]
    B --> B3["活动服务"]
    B --> B4["排行榜服务"]
    B --> B5["邮件/日志服务"]
    B --> C["统一接入"]
    C --> C1["API 网关"]
    C --> C2["认证鉴权"]
    C --> C3["限流路由"]
    B --> D["服务治理"]
    D --> D1["注册发现"]
    D --> D2["配置中心"]
    D --> D3["熔断降级"]
    D --> D4["日志指标链路追踪"]
    B --> E["事件驱动"]
    E --> E1["支付成功事件"]
    E --> E2["活动领奖事件"]
    E --> E3["排行榜变化事件"]
    E --> F["消息队列"]
    F --> G["幂等消费 / 重试 / 补偿 / 最终一致"]
```

### 3.2 答题口令

```text
先按业务边界拆，再用网关和注册发现连，最后用事件驱动解耦。
```

### 3.3 必写 5 点

```text
1. 服务拆分不能按函数拆，要按业务边界和数据归属拆。
2. API 网关负责认证、鉴权、限流、路由和协议适配。
3. 注册发现解决服务扩缩容后的实例变化感知。
4. 可观测性包括日志、指标、链路追踪。
5. 事件驱动要处理可靠投递、幂等消费、失败重试和最终一致性。
```

## 4. 第三优先级：数据缓存高并发

### 4.1 Cache Aside 结构图

```mermaid
flowchart TD
    A["读请求"] --> B["读缓存"]
    B --> C{"缓存命中?"}
    C -->|是| D["返回缓存数据"]
    C -->|否| E["读数据库"]
    E --> F["回填缓存"]
    F --> G["返回数据"]

    H["写请求"] --> I["更新数据库"]
    I --> J["删除缓存"]
    J --> K["后续读请求重新加载"]
```

### 4.2 缓存问题结构图

```mermaid
flowchart TD
    A["缓存异常"] --> B["缓存穿透"]
    A --> C["缓存击穿"]
    A --> D["缓存雪崩"]
    B --> B1["查不存在数据"]
    B --> B2["布隆过滤器 / 空值缓存 / 参数校验"]
    C --> C1["热点 Key 过期"]
    C --> C2["互斥锁 / 逻辑过期 / 热点不过期"]
    D --> D1["大量 Key 同时过期或缓存故障"]
    D --> D2["随机过期 / 预热 / 集群 / 限流降级"]
```

### 4.3 答题口令

```text
读先查缓存，未命中查库并回填；写先改库，再删缓存。
```

### 4.4 必写 5 点

```text
1. Cache Aside 读流程和写流程。
2. 缓存与数据库只能做到最终一致，关键链路要有补偿或校验。
3. 缓存穿透用布隆过滤器和空值缓存。
4. 缓存击穿用互斥锁和热点 Key 保护。
5. 缓存雪崩用随机过期、缓存预热、集群和限流降级。
```

## 5. 第四优先级：质量属性 / 架构评估

### 5.1 质量属性场景六要素图

```mermaid
flowchart LR
    A["刺激源"] --> B["刺激"]
    B --> C["环境"]
    C --> D["制品"]
    D --> E["响应"]
    E --> F["响应度量"]
```

### 5.2 效用树图

```mermaid
flowchart TD
    A["效用 Utility"] --> B["性能"]
    A --> C["可用性"]
    A --> D["安全性"]
    A --> E["可修改性"]
    B --> B1["高并发下 2 秒内响应"]
    C --> C1["节点故障 30 秒内切换"]
    D --> D1["未授权用户不能访问敏感数据"]
    E --> E1["新增设备类型 3 天内完成接入"]
```

### 5.3 风险敏感权衡图

```mermaid
flowchart TD
    A["架构决策"] --> B["敏感点：小变化显著影响质量属性"]
    A --> C["权衡点：提升一个质量属性可能损害另一个"]
    A --> D["风险点：可能导致质量目标无法满足"]
    B --> B1["限流阈值影响可用性和体验"]
    C --> C1["缓存提升性能但增加一致性风险"]
    D --> D1["单数据库导致扩展性和可用性不足"]
```

### 5.4 答题口令

```text
质量属性题先写场景六要素，再写效用树，最后找风险点、敏感点、权衡点。
```

### 5.5 必写 5 点

```text
1. 性能、可用性、安全性、可修改性是最常考质量属性。
2. 质量属性场景必须有刺激源、刺激、环境、制品、响应、响应度量。
3. 效用树从质量属性逐层细化到具体场景。
4. 敏感点是影响质量属性的关键参数或决策。
5. 权衡点是一个决策同时影响多个质量属性。
```

## 6. 第五优先级：DFD / UML / 架构图填空

### 6.1 DFD 图元素

```mermaid
flowchart TD
    A["外部实体"] --> B["加工"]
    B --> C["数据存储"]
    C --> B
    B --> D["外部实体"]
```

### 6.2 DFD 检查口诀

```text
外部实体在系统外；
加工必须有进有出；
数据存储不能直连外部实体；
父图子图输入输出要平衡。
```

### 6.3 UML 选择图

```mermaid
flowchart TD
    A["题目问什么"] --> B{"功能参与者?"}
    B -->|是| C["用例图"]
    A --> D{"类、属性、关系?"}
    D -->|是| E["类图"]
    A --> F{"对象交互顺序?"}
    F -->|是| G["顺序图"]
    A --> H{"对象状态变化?"}
    H -->|是| I["状态图"]
    A --> J{"业务流程活动?"}
    J -->|是| K["活动图"]
    A --> L{"物理部署节点?"}
    L -->|是| M["部署图"]
```

### 6.4 必写 5 点

```text
1. 用例图看参与者和系统功能。
2. 类图看类、属性、方法、关联、聚合、组合、继承。
3. 顺序图看对象生命线和消息顺序。
4. 状态图看状态、事件和迁移。
5. DFD 要检查父子图平衡和加工输入输出。
```

```mermaid
    B --> C["基础软件层 BSW"]

```

### 7.2 边缘计算图

```mermaid
flowchart LR
    A["终端设备"] --> B["边缘节点"]
    B --> C["本地实时处理"]
    B --> D["过滤和聚合数据"]
```

### 7.3 答题口令

```text

边缘计算记四点：低延迟、少带宽、隐私保护、断网可用。
```

## 9. 六个答题口令总表

```text
RAG：先治理知识，再混合检索，最后让大模型基于证据回答。
微服务：先按业务边界拆，再用网关和注册发现连，最后用事件驱动解耦。
缓存：读先查缓存，未命中查库并回填；写先改库，再删缓存。
质量属性：先写场景六要素，再写效用树，最后找风险点、敏感点、权衡点。
DFD/UML：先判断图类型，再检查元素关系和上下文平衡。
AUTOSAR：应用软件层、RTE、基础软件层，靠标准接口实现软硬件解耦。
```
