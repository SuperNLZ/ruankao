# **论软件系统架构风格及其应用**

## 题目

论软件系统架构风格及其应用

## 考场提纲 <span style="font-size: 0.75em; color: #666;">（568 字）</span>

说明：提纲用于考前构思和考场草稿，不计入论文正文。

一、项目背景

- 我参与并设计一款基于Unity引擎的3D手机游戏的开发

二、遇到的问题

- 开发工程终于到了一列的问题，比如：高耦合、项目结构混乱、代码结构不规范

三、架构目标

- 实现一套规范的、高内聚、低耦合的系统架构。使得每个引擎底层代码、网络请求、系统逻辑、美术效果等不同模块之间高度封装，并解耦。

四、关键措施

- 1：梳理代码结构，模块内部高度内聚，对外提供接口；2、使用C#语言的 asmdef 程序集功能，把耦合的代码暴露出来；3、UI 和数据层分离，使用观察者模式，对他们进行解耦

五、实施效果

- 引擎代码高度封装，不会对外依赖。之前美术代码和数据、逻辑代码混在一起，现在逻辑层只管功能逻辑，数据层只负责数据变动，美术效果由事件驱动。网络请求封装在单例类中。使用asmdef把代码打包成程序集，编译速度也大幅度提升

六、总结反思

- 项目开始时应该做好架构设计，避免后续由于代码不规范带来的难以维护和开发低效率

## 摘要 <span style="font-size: 0.75em; color: #666;">（317 字）</span>

我参与设计并设计一款基于 Unity 引擎的3D手游项目。由于项目初期没有做好架构设计，导致在项目开发过程中遇到了不少的问题。项目结构混乱，资源代码目录交错，代码结构不规范，各个模块之间相互调用，耦合度极高。后来采用了三个措施：1、C#的asmdef程序集；2、MVC结构，数据、逻辑、视图分离；3、事件驱动机制，不同模块之前使用事件分发进行消息传递。优化完之后，整个项目结构清爽了很多，开发效率大幅度加快，项目也变得更加容易维护
