## 2. 个人技术背景

日常工作是手机游戏开发工程师，主要技术背景：

- Unity / Cocos 手机游戏客户端开发。
- Python 脚本、工具、数据处理。
- Orleans 服务端开发。
- MySQL 数据库。

论文和案例中应将个人经验包装为架构项目，而不是单纯写客户端开发。

统一项目背景：

> 面向移动游戏的高并发在线服务平台。

项目可包含以下模块：

- 账号登录
- 角色管理
- 匹配组队
- 排行榜
- 商城支付
- 活动配置
- 邮件公告
- 数据统计
- 运营后台

项目核心技术表达：

| 实际经验 | 考试表达 |
|---|---|
| Unity / Cocos | 多端客户端、移动端接入、弱网重连、协议兼容、版本灰度 |
| Orleans | 分布式服务框架、Actor 模型、状态封装、并发隔离、横向扩展 |
| MySQL | 关系型数据库、事务一致性、索引优化、读写分离、冷热数据分离 |
| Python | 自动化运维、数据处理、运营工具、批处理任务 |
| 游戏活动 | 高并发峰值、弹性扩容、缓存预热、限流降级 |
| 账号/支付/道具 | 安全、事务、幂等、审计、数据一致性 |
| 排行榜/匹配 | 缓存、异步处理、性能优化、实时性权衡 |

## 5. 论文六大主题

所有论文统一套用“面向移动游戏的高并发在线服务平台”项目背景。

### 5.1 架构风格

题目方向：

- 论软件系统架构风格及其应用。

可写内容：

- C/S 架构。
- 分层架构。
- 微服务架构。
- Actor 模型。
- 服务端负责核心状态和业务规则，客户端负责表现和交互。

### 5.2 高可用架构

题目方向：

- 论高可用架构设计。
- 论软件系统可靠性设计。

可写内容：

- 服务集群。
- 登录、支付、活动服务故障隔离。
- 限流、降级、熔断。
- MySQL 备份恢复。
- 活动峰值流量应对。

### 5.3 性能优化

题目方向：

- 论系统性能优化设计。
- 论高并发系统设计。

可写内容：

- 缓存热点数据。
- 异步处理日志、邮件、活动结算。
- MySQL 索引优化。
- 排行榜、匹配服务优化。
- 活动开启前缓存预热。

### 5.4 安全架构

题目方向：

- 论系统安全架构设计。

可写内容：

- 账号认证。
- Token 和接口签名。
- 防重放。
- 支付回调校验。
- 权限控制。
- 日志审计。

### 5.5 数据架构

题目方向：

- 论数据架构设计。
- 论数据库设计及优化。

可写内容：

- MySQL 存储账号、角色、订单、道具等核心数据。
- 事务一致性。
- 读写分离。
- 冷热数据分离。
- 日志数据异步入库。

### 5.6 微服务架构

题目方向：

- 论微服务架构及其应用。

可写内容：

- 账号服务、角色服务、商城服务、活动服务、排行榜服务、运营后台服务拆分。
- 服务注册与发现。
- 接口协议治理。
- 服务自治。
- 部署和扩容。

## 6. 论文母版结构

每篇论文统一使用以下结构：

1. 摘要
2. 项目背景
3. 面临的问题
4. 架构设计目标
5. 关键架构方案
6. 实施效果
7. 总结反思

摘要建议覆盖：

- 我参与了什么项目。
- 项目遇到什么问题。
- 采用了什么架构或技术措施。
- 取得了什么效果。

正文必须体现：

- 架构目标。
- 技术选型。
- 质量属性。
- 取舍分析。
- 实施结果。

必须避免：

- 只写 Unity 或 Cocos 客户端开发。
- 只堆技术名词，不结合项目问题。
- 只写功能，不写架构质量属性。
- 没有实施效果和反思。

## 7. 案例分析答题框架

案例题统一按以下套路组织答案：

1. 先指出问题。
   - 性能瓶颈。
   - 单点故障。
   - 耦合过高。
   - 数据不一致。
   - 安全风险。

2. 再给解决方案。
   - 缓存。
   - 异步处理。
   - 服务拆分。
   - 限流降级。
   - 数据库优化。
   - 权限控制。

3. 再说明效果。
   - 提升可用性。
   - 提升扩展性。
   - 提升性能。
   - 提升安全性。
   - 提升可维护性。

4. 如果问原因，从质量属性回答。

5. 如果问取舍，说明一致性、性能、复杂度、成本之间的权衡。

每道案例题复盘格式：

```text
题目来源：
题目类型：
标准答案关键词：
我漏掉的点：
可复用答题句：
下次遇到类似题的判断方法：
```

## 11. 判断复习是否有效的标准

每天结束前检查：

- 今天是否碰了论文。
- 今天是否做了真题。
- 今天是否记录了错题。
- 今天是否沉淀了可复用答题句。

如果四项缺一项，第二天优先补缺失项。

考前最低完成标准：

- 6 个论文主题模板。
- 3 篇限时论文全文。
- 2020-2022 真题至少完成综合和案例核心部分。
- 2024-2025 回忆版至少完成趋势浏览和错题记录。
- 案例分析形成 20 条以上可复用答题句。
- 综合错题形成 80 条以上考场记忆结论。
